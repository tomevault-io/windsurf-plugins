# Regras de Frontend (React)

- **Linguagem:** Escreva código TypeScript limpo e idiomático.
- **Tipagem:** Utilize tipos explícitos sempre que possível para aproveitar ao máximo a segurança de tipos do TypeScript.
- **Estilo de Programação:** Prefira o uso de funções puras e padrões de imutabilidade para evitar efeitos colaterais e facilitar o gerenciamento de estado.
- **React:** Use a sintaxe moderna do React, com componentes funcionais e Hooks (`useState`, `useEffect`, `useContext`, etc.).
- **Código Assíncrono:** Manipule operações assíncronas exclusivamente com `async/await` para um código mais legível e linear.
- **Validação:** Prefira a biblioteca Zod para validação de esquemas (schemas) e dados, garantindo que os dados que fluem pela aplicação estejam corretos.
