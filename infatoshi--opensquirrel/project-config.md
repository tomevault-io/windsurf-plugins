# OpenSquirrel

GPU-rendered, keyboard-driven agent tiling manager built on GPUI (Rust).

## REOPEN / RELAUNCH THE APP

When the user says "reopen", "open the app", "show it to me", "relaunch", "rebuild and open", "close and reopen", or anything meaning "I want to see the running app", run this EXACT command as a single Bash call:

```bash
pkill -f 'OpenSquirrel-bin' 2>/dev/null; sleep 0.3 && cargo build --release 2>&1 | tail -3 && cp target/release/opensquirrel dist/OpenSquirrel.app/Contents/MacOS/OpenSquirrel-bin && open dist/OpenSquirrel.app
```

Do NOT use `cargo run`. Always kill, build release, copy into the .app bundle, and `open` the bundle.

## Build / Run / Test

```bash
cargo build --release              # release build
cargo test                         # run all tests
```

## Key Constraints

- Rust 2024 edition (>=1.85), gpui 0.2 standalone crate (not a Zed fork)
- macOS (Metal) primary, Linux (Vulkan x86_64) compiles and tests pass
- Code is modular: app.rs (~8K lines), config.rs, runtime.rs, lib.rs, theme.rs, hooks.rs, changes.rs, worktree.rs
- Use offset/limit when reading app.rs, or grep for specific sections
- lib.rs contains line classification, markdown span parsing, diff summary utils
- Agents are subprocesses (claude CLI, cursor CLI, codex CLI, opencode CLI, gemini CLI) -- not an agent runtime
- SPEC.md is the north star for features and phasing

## Config / Data directory

`~/.osq/` (not ~/.opensquirrel -- was renamed). Contains:
- `config.toml` -- runtimes, machines, MCPs, theme, font, settings
- `state.json` -- agents, transcripts, scroll positions, worker assignments
- `bin/` -- agent wrapper scripts (generated by hooks server)
- `worktrees/` -- git worktrees for isolated agents

## .app bundle structure (dist/)

- `Contents/MacOS/OpenSquirrel` -- wrapper shell script that sources ~/.zshrc for PATH
- `Contents/MacOS/OpenSquirrel-bin` -- the actual binary (copied from cargo build)
- `Contents/Resources/OpenSquirrel.icns` -- app icon
- `Contents/Info.plist` -- bundle metadata
