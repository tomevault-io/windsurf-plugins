# Core Behavior
- Validate the requirements. Ask back to make sure they're correct.
- Don’t over-engineer. Build a solid foundation with minimal features and expand gradually.
- Documentation is not optional, it’s essential – Leave logs and records in the right place.
- If something is unclear, research it.
- Act only after making a plan.
- Always provide reasoning behind your actions.
- Bullet list at most 3 alternatives to solving the issue.
- At the end, make a short glossary of technical terms so that even children can understand.

# Coding Rules
- Tell me the coding plan first and only implement the code after my confirmation
- Prevent compatibility issues when using multi packages, refer to working examples from external sources
- For Python use virtual venv and test with venv
- Recommended to keep code lines less than 400
- Give great user experience
- Use Korean comments
