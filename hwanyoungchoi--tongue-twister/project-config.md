# AI 코딩 가이드라인

## 1. 프로젝트 개요

- **목표**: 모바일 웹 기반의 고퀄리티 젠말놀이 게임 구현.
- **핵심 가치**: 높은 수준의 UI 연출, 부드러운 애니메이션, 사용자 중심의 간결한 흐름.

## 2. 디자인 시스템 규칙 (Strict)

- **폰트**: 타이틀 및 강조 텍스트에 `font-np` 필수 적용. 기본적으로 `pretendard` 사용.
- **인터랙션**:
  - 버튼 Hover: `scale-[0.99]` (디자인 메모 준수).
  - Lottie: 동적인 UI 피드백 및 애니메이션에 활용.
- **컬러**:

## 3. 핵심 로직 흐름 (Game Flow)

1. **초기화**: `AppEntry`에서 2.5초 스플래시 노출 및 폰트 로딩 대기. 앱 실행 초반 레이아웃 내에서 사전 필요한 로직 이곳에 작성.
2. **설정**: 2~10명 인원 설정 및 닉네임 입력 (닉네임 없을 시 기본값 부여).
3. **모드**: '타이머 모드'(제한시간 내 읽기)와 '양심 모드'(성공여부 자율판단) 분기.
4. **벌칙**: 게임 종료 후 `finish` 컴포넌트에서 관리하며, 설정된 벌칙이 있을 경우 노출 후 랭킹 화면으로 전환. 벌칙 미설정 시에는 랭킹 화면으로 바로 이동.

## 4. 상세 게임 메커니즘 (Game Logic)

### 공통 설정

- **인원**: 10명.
- **구성**: 총 3라운드 (라운드당 10장, 총 30장 카드 소진).
- **벌칙**: 시작 전 직접 입력 또는 리스트 선택 가능, 최종 탈락자에게 자동 적용.

### 모드 1: ⏱ 타이머 모드 (시간초 제한 모드, 통과 버튼만 있음)

- **로직**: 각 카드마다 제한 시간 설정 (ex. 3초). 플레이어는 주어진 시간 내에 카드 문장을 정확히 읽고 '통과' 버튼을 클릭해야 합니다. 시간 내 통과하지 못하면 자동으로 목숨 -1.
- **패널티**: 1인당 목숨은 2개, 목숨 2개 이상 소진 시 자동 탈락.
- **점수/승패**: 성공 시 각 라운드의 남은 시간 초가 점수로 환산됩니다 (+n점 안내). 게임 종료 후, 탈락하지 않은 플레이어 중 남은 초가 가장 많은 플레이어가 1등 (동점 시 공동 순위). 탈락자가 많을 경우 벌칙 처리는 탈락자 중 가장 먼저 탈락한 사람에게.
- **특수 카드**:
  - **보너스**: 시간 추가/멈춤, 턴 건너뛰기, 목숨 추가.
  - **벌칙**: 시간 감소, 카드 흔들림/블러, 두 장 읽기, 방해 요소(먼지/눈뽕) 등장.

### 모드 2: ⏳ 양심 모드 (?) (시간초 없는 모드, 성공/실패 버튼 있음)

- **로직**: 시간 제한 없이 문장을 읽은 후, 플레이어가 직접 '성공 / 실패' 버튼을 누릅니다. 성공/실패 판단은 자율이며, 정직한 게임을 전제로 합니다.
- **패널티**: 실패 시 1벌점, 누적 실패 2회 시 탈락. 라운드마다 실패 횟수 기록.
- **점수/승패**: 3라운드 종료 후 가장 많은 실패 횟수를 기록한 플레이어가 꼴찌. 동률일 경우 벌칙은 랜덤 또는 미리 정한 룰에 따라 진행.
- **특수 카드**: 턴 건너뛰기, 목숨 추가, 카드 흔들림, 틀렸던 카드 재등장.

## 5. 개발 및 답변 지침

- **답변 스타일**: 모든 설명은 핵심을 먼저 말하는 **두괄식**으로 작성할 것.
- **컴포넌트**:
  - `components/ui`: 기능 위주의 순수 컴포넌트 (스타일 최소화).
  - `components/common`: 프로젝트 디자인 명세가 반영된 확장 컴포넌트 (GameButton, GameSwitch 등).
- **스택**: React + Vite 기반 (CSR), Tailwind CSS 스타일링 적용. 단일 페이지(Single Page Application) 구조로 개발하며, 게임 컴포넌트 내의 모든 화면 전환은 `react-router-dom`을 사용하여 처리함.
