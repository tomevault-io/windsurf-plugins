# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

**Sourcebook** is an architecture-first development tool: developers design systems visually on a canvas, and an AI coding agent uses the diagram as its implementation spec. See `CONCEPT.md` for the full vision.

## Tech Stack

| Layer | Technology |
|---|---|
| Frontend | SvelteKit (TypeScript) |
| Backend | Python FastAPI |
| Database | SQLite (local persistence) |
| Code Analysis | Tree-sitter (multi-language parsers) — stubbed |
| AI Integration | `claude` CLI via subprocess (`--print --output-format stream-json`) |
| Communication | WebSocket (real-time, bidirectional) |
| Canvas | Custom SVG renderer with pan/zoom/drag |

## Architecture

### Single-Process Application

The SvelteKit frontend is compiled to a static SPA and served by FastAPI. One port, one process.

```
Browser → http://localhost:8000
               │
         [FastAPI :8000]
           ├── GET  /        → serves built SvelteKit SPA (backend/static/)
           ├── GET  /health  → health check
           ├── WS   /ws/{id} → WebSocket handler
           ├── AI Agent (Claude CLI subprocess)
           ├── Static Analysis (Tree-sitter)
           └── Persistence (SQLite)
```

### Frontend (SvelteKit)
- **Canvas**: Pannable/zoomable diagram surface — nodes = modules/components, edges = dependencies/data flow
- **Chat Pane**: Sidebar chat connected to the AI agent via WebSocket
- **State**: Canvas state (nodes, edges, human overrides, annotations) is the core app state

### Backend (FastAPI)
- **WebSocket endpoint**: Persistent connection for streaming AI responses and real-time diagram updates
- **AI Agent**: Spawns `claude --print --output-format stream-json` as an async subprocess; streams token events back over WebSocket
- **Static Analysis Service**: Tree-sitter parsers extract code structure to keep the diagram in sync
- **Persistence**: SQLite stores diagram layout, human overrides, generated code mappings, and analysis cache

### Core Data Model Concepts
- **Nodes**: Represent modules, components, services, data models
- **Edges**: Represent dependencies, data flow, API calls
- **Overrides**: Human corrections to AI-inferred elements — stored as ground truth, always take precedence over re-parsed inference
- **Diagram-as-Prompt**: The serialized canvas state is the primary context sent to the AI agent for code generation

## Key Principles (from CONCEPT.md)

1. **Architecture First**: The diagram is the source of truth — design before code
2. **Living Documentation**: Static analysis keeps the diagram in sync with actual code; drift is surfaced to the developer
3. **Human-in-the-Loop**: AI proposes, human overrides, overrides persist and override AI inference on future re-parses

## Development Commands

```bash
# One-time setup
make install         # npm install + pip install -e ".[dev]"

# Run everything (builds frontend, starts backend on :8000)
make dev             # build + uvicorn --reload
make start           # build + uvicorn (production)
make build           # frontend build only → backend/static/

# Frontend (from frontend/)
npm run check        # svelte-check type checking
npm run lint         # eslint + prettier

# Backend (from backend/)
pip install -e ".[dev]"
pytest
ruff check .
ruff format .
```

## Key Files

| File | Purpose |
|---|---|
| `Makefile` | Root-level commands: `make dev`, `make install`, `make build` |
| `backend/app/main.py` | FastAPI app, WebSocket route, serves `backend/static/` via StaticFiles |
| `backend/app/ws_handler.py` | WebSocket message loop, AI streaming, diagram persistence |
| `backend/app/ai_agent.py` | Spawns claude CLI subprocess, builds prompt (system+diagram+history), parses stream-json |
| `backend/app/database.py` | SQLite schema, diagram CRUD, node override persistence |
| `backend/static/` | Built SvelteKit SPA — generated by `make build`, served by FastAPI |
| `frontend/src/routes/+layout.ts` | Disables SSR (`ssr=false`) so SvelteKit builds as a static SPA |
| `frontend/src/lib/ws.ts` | WebSocket client — derives URL from `window.location` (same-origin) |
| `frontend/src/lib/stores/diagram.ts` | Diagram state (nodes/edges) as a Svelte writable store |
| `frontend/src/lib/components/Canvas.svelte` | SVG canvas with pan/zoom/drag |
| `frontend/src/lib/components/ChatPane.svelte` | Chat UI connected to WS |
