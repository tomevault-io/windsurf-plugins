# Expo/React Native Project Rules

## Project Context
Expo/React Native application with web support and TypeScript.

## Code Style
- TypeScript: 4 spaces
- Follow `.editorconfig`

## Component Patterns
- React Native components: `View`, `Text`, `ScrollView`, etc.
- Use Expo APIs for native functionality
- Platform-specific code with `Platform.OS`
- Responsive design for multiple screen sizes

## File Organization
- Components: `components/` or `src/components/`
- Screens: `screens/` or `app/`
- Utils: `utils/` or `src/utils/`
- Assets: `assets/`

## Best Practices
- Test on both iOS and Android
- Use Expo managed workflow features
- Handle platform differences gracefully
- Optimize for mobile performance
- Use proper TypeScript types
- Keep components focused and reusable
