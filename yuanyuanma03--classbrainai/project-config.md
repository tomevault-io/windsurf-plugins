## Project Overview

ClassBrainAI is a multi-agent collaboration system for intelligent class management, adapted from the edict architecture (which was based on the ancient Chinese "Three Provinces and Six Ministries" system). It uses a school class as the thematic metaphor, with AI agents representing various class committee positions (Monitor, Study Committee, Chair, Vice Monitor, etc.).

**Key Concept:** Tasks flow through a pipeline: Pending → Monitor → Study → Chair → Assigned → Doing → Review → Done

## Common Development Commands

### Starting the Dashboard
```bash
# Start the dashboard server (port 7891 by default)
cd dashboard
python server.py

# Or with custom port
python server.py --port 7892

# Access at http://127.0.0.1:7891
```

### Task Management (class_board.py)
```bash
# Create a new task
python3 scripts/class_board.py create CB-20260322-001 "Task title" Monitor 班长 班长

# Update task state
python3 scripts/class_board.py state CB-20260322-001 Study "Transfer to Study Committee"

# Add flow log entry
python3 scripts/class_board.py flow CB-20260322-001 "班长" "学习委员" "Task transferred"

# Report progress (without changing state)
python3 scripts/class_board.py progress CB-20260322-001 "Analyzing requirements" "Completed✅|In Progress🔄|Planned"

# Mark task as done
python3 scripts/class_board.py done CB-20260322-001 "/path/to/output" "Completion summary"

# Update todo items
python3 scripts/class_board.py todo CB-20260322-001 1 "Implement API" in-progress
```

## Architecture

### Agent System (Multi-Agent Collaboration)

The system uses 11 AI agents, each with a SOUL.md file defining their personality and responsibilities:

| Agent | Directory | Responsibility |
|-------|-----------|----------------|
| 班长 (Monitor) | `agents/monitor/` | Message triage, initial judgment, task creation |
| 学习委员 (Study) | `agents/study/` | Planning, breaking down tasks |
| 班委主席 (Chair) | `agents/chair/` | Review and approval |
| 副班长 (Vice Monitor) | `agents/vice_monitor/` | Task dispatching |
| 技术委员 (Technical) | `agents/technical/` | Code and technical tasks |
| 后勤委员 (Logistics) | `agents/logistics/` | Infrastructure and tools |
| 财务委员 (Finance) | `agents/finance/` | Data and resources |
| 文艺委员 (Arts) | `agents/arts/` | Documentation and knowledge |
| 纪律委员 (Discipline) | `agents/discipline/` | Quality and standards |
| 组织委员 (Organization) | `agents/organization/` | Configuration and assignment |
| 早读委员 (Morning Reading) | `agents/morning_reading/` | Daily information collection |

### State Machine

Task states follow a strict transition flow:
- `Pending` → `Monitor` → `Study` → `Chair` → `Assigned` → `Doing` → `Review` → `Done`
- `Blocked` can be entered from most states and can transition to `Doing` or `Cancelled`

See `_VALID_TRANSITIONS` in `scripts/class_board.py` for the complete rules.

### Data Storage

- **Primary data:** `data/tasks_source.json` - All task data
- **Derived data:** `data/live_status.json` - Generated by `refresh_live_data.py`
- **Concurrency:** Uses file locking (`scripts/file_lock.py`) for atomic operations

### Dashboard Architecture

- **Backend:** Python HTTP server using standard library (`dashboard/server.py`)
- **Frontend:** Single-file HTML/CSS/JavaScript (`dashboard/dashboard.html`)
- **API:** RESTful endpoints for live status, agent config, and task management

## Important Conventions

### Task ID Format
- Format: `CB-YYYYMMDD-NNN` (e.g., `CB-20260322-001`)
- `CB` = ClassBrain prefix
- NNN increments daily starting from 001

### File Lock Usage
Always use the atomic operations from `file_lock.py` when modifying `tasks_source.json`:
- `atomic_json_read()` - Read with shared lock
- `atomic_json_update()` - Read-modify-write with exclusive lock
- `atomic_json_write()` - Write with exclusive lock

## Development Notes

### When Modifying Agent Behavior
1. Edit the corresponding `SOUL.md` file in `agents/*/`
2. The SOUL.md defines the agent's personality and decision rules
3. Agents are invoked externally (not part of this codebase's runtime)

### When Adding New Task States
1. Update `STATES` list in `scripts/class_board.py`
2. Update `_VALID_TRANSITIONS` dict for state flow rules
3. Update `STATE_ORG_MAP` for agent mappings
4. Update frontend state displays in `dashboard/dashboard.html`

### When Modifying Dashboard UI
- The dashboard uses a campus color scheme (blue/cyan theme)
- CSS variables are defined in `<style>` section of `dashboard.html`
- State colors follow the pattern `.st-StateName`
