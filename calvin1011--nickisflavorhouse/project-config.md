# No Em-Dashes in User-Facing Text

Do not use em-dashes (`—`, `&mdash;`, or ` — `) in any string shown to users (UI labels, headings, descriptions, email subjects, notification text, button text, placeholder text).

Use one of these instead:

- **Colon** for label/value pairs: `Cash App: $nickiydoll`, `Zelle: email@example.com`
- **Comma** to join related clauses: `Paid in full, $25.00`
- **Period** to separate sentences: `Reach out through Contact. I'm here to help.`
- **Pipe** for compact separators in push/SMS: `.join(' | ')`

Em-dashes in **code comments** (JSDoc, inline dev notes) are acceptable.
