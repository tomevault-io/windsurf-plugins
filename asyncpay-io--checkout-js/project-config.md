# asyncpay-checkout-sdk — Claude AI Assistant Context

## Project Overview

`@asyncpay/checkout` is a TypeScript SDK that embeds a hosted checkout iframe into any web page, connecting it to the Asyncpay payment orchestration platform. A single integration gives merchants access to Stripe, Paystack, Flutterwave, Fincra, and PayPal — with no per-gateway code on the frontend.

**npm:** `@asyncpay/checkout`
**GitHub:** https://github.com/Asyncpay-io/checkout-js
**Current version:** `0.0.38`
**Demo:** https://asyncpay.io/demo

## How the SDK Works

1. **Caller invokes `AsyncpayCheckout(options)`** — This is the only public export.
2. **Input validation** — `customerEmail`, `customerUUID`, or a full `customer` object must be present. Amount and currency are validated unless a `subscriptionPlanUUID` or `subscriptionPlanLink` is given. `metadata` keys/values must all be strings.
3. **API call** — A `POST` to `/v1/sdk/initialize-payment-request` is made with the merchant's `publicKey` in the `Authentication: Bearer` header.
4. **Redirect vs iframe** — If `body.data.should_redirect` is true, the page navigates directly. Otherwise, a full-screen fixed overlay iframe is injected into `document.body`.
5. **`postMessage` event bridge** — The iframe posts `eventType` messages (`closeIframe`, `showLoader`, `paymentSuccessful`, `redirect`) back to the parent page. The SDK handles these and fires the appropriate callbacks.
6. **Cleanup** — `unsetCheckoutSession()` removes the iframe from the DOM and clears the `sessionStorage` lock key.

## Key Options

| Option | Type | Notes |
|---|---|---|
| `publicKey` | `string` | **Required.** Business public key from the Asyncpay dashboard. |
| `amount` | `number \| string` | Required unless `subscriptionPlanUUID`/`subscriptionPlanLink` is set. |
| `currency` | `string` | ISO 4217 3-letter code (e.g. `NGN`, `USD`). |
| `subscriptionPlanUUID` | `string` | Routes checkout to a subscription flow; ignores `amount`/`currency`/`description`. |
| `subscriptionPlanLink` | `string` | Alternative to UUID for subscription routing. |
| `extendSubscriptionBy` | `string \| number` | Number of billing periods to extend an existing subscription. Sent as `extend_subscription_by` (integer). |
| `metadata` | `Record<string, string>` | Sent as `meta_data`. All keys and values must be strings. |
| `savePaymentMethod` | `boolean` | Saves the payment method used for future off-session charges. |
| `reference` | `string` | Merchant-supplied idempotency reference. |
| `environment` | `"prod" \| "dev" \| "local"` | Defaults to `"prod"`. Use `"local"` to hit `http://localhost`. |

## Build & Dist

```bash
npm run build   # runs: rollup --config
```

Produces three bundles:

| File | Format | Use case |
|---|---|---|
| `dist/bundle.js` | CommonJS | Node / legacy bundlers |
| `dist/bundle.esm.js` | ES Module | Modern bundlers (Vite, webpack 5, Rollup) — tree-shakeable |
| `dist/bundle.umd.js` | UMD | `<script>` tag in HTML, exposes `window.AsyncpayCheckout` |

The `dist/` directory is **committed to the repository** and published to npm as-is (see `"files": ["dist"]` in `package.json`).

## Entry Points in package.json

```json
"main":    "dist/bundle.js"      // CJS — Node / legacy require()
"module":  "dist/bundle.esm.js"  // ESM — modern bundler import
"browser": "dist/bundle.umd.js"  // UMD — script tag
"types":   "dist/index.d.ts"     // TypeScript declarations
```
