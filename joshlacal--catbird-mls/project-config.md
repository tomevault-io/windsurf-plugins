# catbird-mls

## Overview

`catbird-mls` is the core MLS (Messaging Layer Security) SDK for all Catbird clients. It implements OpenMLS 0.8 group cryptography, a platform-agnostic orchestrator (sync, recovery, messaging policy), and UniFFI bindings so Swift/Kotlin call into a single Rust library. It also contains voice encoding (Opus/OGG), blob crypto (AES-256-GCM), and a WASM client facade — making it a monolith that should eventually be split. The spec that governs all behavior is `docs/MLS_CLIENT_PROTOCOL.md`.

## Build & Test Commands

```bash
# Standard build
cd catbird-mls && cargo build

# Tests
cargo test
cargo test --test two_user_messaging_test -- --nocapture
cargo test --test persistence_repro -- --nocapture
cargo test --test member_removal_test -- --nocapture

# Lint
cargo fmt --check && cargo clippy --all-targets --all-features

# Build for WASM (excludes SQLite, tokio rt-multi-thread, UniFFI)
cargo build --target wasm32-unknown-unknown
```

### iOS XCFramework

```bash
# Full build + Swift bindings + XCFramework
./create-xcframework.sh

# Or build targets only (no XCFramework packaging)
./build-ios.sh
```

Swift bindings generated by in-crate `uniffi-bindgen`:
```bash
cargo run --bin uniffi-bindgen generate \
    --library target/<HOST>/release/libcatbird_mls.dylib \
    --language swift --out-dir build/bindings --config uniffi.toml
```

### Android JNI

```bash
./build-android.sh [--android-module-dir ../android/Catbird/mlsffi]
```
Reads NDK from `~/Library/Android/sdk/ndk` (auto-detected). Linkers configured in `.cargo/config.toml`.

## Source Map

All source is in `src/`. Single crate, no workspace.

### Orchestrator (the core)

| File | Purpose |
|------|---------|
| `orchestrator/orchestrator.rs` | `MLSOrchestrator<S,A,C,M>` — generic core struct, owns all in-memory state |
| `orchestrator/storage.rs` | `MLSStorageBackend` trait — 20 persistence methods, default no-ops for backward compat |
| `orchestrator/api_client.rs` | `MLSAPIClient` trait — 20+ DS/nest HTTP methods |
| `orchestrator/credentials.rs` | `CredentialStore` trait — signing key, MLS DID, device UUID (9 methods) |
| `orchestrator/mls_provider.rs` | `MlsCryptoContext` trait — synchronous OpenMLS operations (20 methods) |
| `orchestrator/constants.rs` | **Single source of truth** for all spec §10 constants. Never hardcode elsewhere. |
| `orchestrator/types.rs` | Shared types: `ConversationView`, `Message`, `GroupState`, `MLSMessagePayload` |
| `orchestrator/groups.rs` | `create_group`, `leave_group`, `reset_group` |
| `orchestrator/messaging.rs` | `send_message`, `send_reaction`, `fetch_messages`, `process_incoming_message` |
| `orchestrator/sync.rs` | `sync_with_server` — circuit breaker, pagination, conversation reconciliation |
| `orchestrator/recovery.rs` | `RecoveryTracker`, `SequencerFailoverTracker`, fork-detection helpers |
| `orchestrator/key_packages.rs` | Key package publishing and replenishment |
| `orchestrator/devices.rs` | `ensure_device_registered` — credential generation |
| `orchestrator/ordering.rs` | `OrderingState` — gap detection for sequence numbers |
| `orchestrator/error.rs` | `OrchestratorError` with `is_remote_data_error()`, `is_rate_limited()` |

### UniFFI Bridge

| File | Purpose |
|------|---------|
| `catbird_mls.rs` | `CatbirdMls` — the UniFFI-exported entry point Swift/Kotlin calls |
| `orchestrator_bridge.rs` | Callback interfaces (`OrchestratorStorageCallback`, `OrchestratorAPICallback`, `OrchestratorCredentialCallback`) + adapter structs |
| `api.rs` | `MLSContext` UniFFI object — direct MLS crypto operations, traffic-analysis-resistant padding |
| `bridge_mappers.rs` | Conversions between internal types and FFI records |
| `types.rs` | FFI-safe records: `KeyPackageData`, `GroupCreationResult`, `DecryptResult`, etc. |
| `error.rs` | `MLSError` — top-level UniFFI `flat_error` enum |

### MLS Core

| File | Purpose |
|------|---------|
| `mls_context.rs` | `MLSContext` inner impl — SQLCipher connection + `MlsGroup` HashMap. All OpenMLS calls. |
| `epoch_storage.rs` | `EpochSecretManager` — exports/stores epoch secrets for forward secrecy |
| `hybrid_storage.rs` | `HybridStorageProvider` — wraps `openmls_sqlite_storage` + keychain (currently unused) |
| `group_metadata.rs` | `GroupMetadata` + extension type `0xff00` — encrypted name/description in MLS context |

### Client Facades

| File | Purpose |
|------|---------|
| `client.rs` | `CatbirdClient<S,A,C,M>` — MLS-free chat facade for catmos desktop |
| `client_bridge.rs` | Bridges `CatbirdClient` to UniFFI |
| `wasm_client.rs` | `WasmClient` — same interface with `?Send` async traits for wasm32 |
| `high_level_contract.rs` | Request/response structs for the easy-mode facade |
