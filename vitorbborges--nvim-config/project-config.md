# Neovim Configuration

## Project Overview

This repository contains a personal Neovim configuration, meticulously crafted in Lua. It leverages the `lazy.nvim` plugin manager to create a modern, modular, and feature-rich terminal-based editing experience.

The configuration is structured logically:
- `init.lua`: The main entry point that loads the core settings and plugins.
- `lua/vitor/core/`: Contains fundamental editor settings, such as `options.lua` and `keymaps.lua`.
- `lua/vitor/plugins/`: Each file in this directory corresponds to a specific plugin's configuration, allowing for clean separation of concerns.
- `lazy-lock.json`: Automatically generated by `lazy.nvim` to lock plugin versions for reproducible setups.

## Key Features & Plugins

This Neovim setup is built upon a collection of powerful plugins:

- **Plugin Manager**: [lazy.nvim](https://github.com/folke/lazy.nvim) for declarative and fast plugin loading.
- **LSP & Completion**:
    - [nvim-lspconfig](https://github.com/neovim/nvim-lspconfig) for configuring Language Server Protocols.
    - [mason.nvim](https://github.com/williamboman/mason.nvim) to manage LSP servers, linters, and formatters.
    - [nvim-cmp](https://github.com/hrsh7th/nvim-cmp) for autocompletion.
- **Fuzzy Finding**: [telescope.nvim](https://github.com/nvim-telescope/telescope.nvim) is heavily customized for finding files, projects, text (`live_grep`), and more.
- **Syntax Highlighting**: [nvim-treesitter](https://github.com/nvim-treesitter/nvim-treesitter) provides fast and accurate, language-aware highlighting, indentation, and text objects.
- **File Navigation**: [nvim-tree.lua](https://github.com/nvim-tree/nvim-tree.lua) serves as the file explorer.
- **Aesthetics & UI**:
    - **Colorscheme**: [catppuccin](https://github.com/catppuccin/nvim) (`mocha` flavor).
    - **Statusline**: [lualine.nvim](https://github.com/nvim-lualine/lualine.nvim).
    - **Tab/Buffer Line**: [bufferline.nvim](https://github.com/akinsho/bufferline.nvim).
- **Git Integration**: [gitsigns.nvim](https://github.com/lewis6991/gitsigns.nvim) adds Git decorations to the sign column.
- **Quality of Life**: Includes plugins for auto-pairing characters, managing comments, surrounding text with delimiters, and more.

## Building and Running

This is a configuration, not a standalone application, so there is no "build" process.

### Running

Simply launch Neovim:
```bash
nvim
```

### Plugin Management

The `lazy.nvim` plugin manager handles all plugin-related operations. Common commands are executed from within Neovim:

- **Open the Lazy UI**: `:Lazy`
- **Sync/Install Plugins**: `:Lazy sync`
- **Update Plugins**: `:Lazy update`
- **Check for Issues**: `:Lazy health`

## Development Conventions

### Configuration Structure

- **Core Settings**: General editor behavior is defined in `lua/vitor/core/options.lua`.
- **Keymappings**: Custom shortcuts are set in `lua/vitor/core/keymaps.lua`. The leader key is `Space`.
- **Adding a New Plugin**: To add a plugin, create a new `.lua` file in the `lua/vitor/plugins/` directory. The file should return a `lazy.nvim` plugin specification. `lazy.nvim` will automatically detect and load it.

### Code Style

The project uses [StyLua](httpss://github.com/JohnnyMorganz/StyLua) for formatting Lua code. The configuration can be found in `.stylua.toml`.
