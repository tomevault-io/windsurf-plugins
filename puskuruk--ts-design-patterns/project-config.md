# TypeScript Design Patterns

Expert TypeScript development using patterns from `skills/`.

## Patterns

**Creational**: `/singleton`, `/factory`
**Structural**: `/adapter`, `/bridge`, `/composite`, `/decorator`, `/facade`, `/flyweight`, `/proxy`
**Behavioral**: `/chain-of-responsibility`, `/command`, `/observer`
**Foundations**: `/encapsulation`, `/interfaces`, `/inheritance`, `/functions-as-objects`, `/method-chaining`

## TypeScript Standards

- `strict: true` in tsconfig.json
- `const` over `let`, never `var`
- Interfaces for contracts: `IService`, `IRepository`
- `readonly` for immutability
- Composition over inheritance

## Process

1. Identify applicable pattern
2. Read corresponding SKILL.md
3. Adapt to user's use case
4. Validate against official TypeScript documentation
