## 🎯 Project Overview

**yt-gemini-summarizer** is a premium YouTube transcript fetcher and AI summarizer designed to work seamlessly with Gemini CLI in VS Code. It allows users to:

1. Fetch transcripts from any YouTube video
2. Generate stunning, well-structured summaries using Gemini AI
3. Use the custom `/ytsum` slash command for instant summaries

## 🚀 How to Use /ytsum Command

When a user types `/ytsum <url>`, you should:

### Step 1: Check if transcript exists
Look for the transcript file in:
- `output/transcript-{VIDEO_ID}.txt`
- `transcripts/transcript-{VIDEO_ID}.txt`

### Step 2: If transcript doesn't exist
Run the CLI tool to fetch it:
```bash
node ytsum.js <url>
```

### Step 3: Read the transcript
Read the full contents of the transcript file.

### Step 4: Generate the summary
Create a premium summary following the exact format below.

## 📏 Formatting Rules

### Timestamps
- Always format as `[MM:SS]` or `[HH:MM:SS]` for longer videos
- Make timestamps clickable: `[5:30](https://youtube.com/watch?v=ID&t=330)`
- Include timestamps at natural topic transitions

### Code Blocks
- Use proper language identifiers: ```javascript, ```python, ```bash
- Include full, runnable code when possible
- Add comments explaining complex parts

### Quotes
- Use exact words from the transcript when possible
- Always include the timestamp
- Bold or emphasize key phrases

### Key Points
- Be specific, not vague
- Include numbers and details
- Make them actionable when possible

### Tables
- Use tables for comparisons, timelines, and reference cards
- Keep column headers concise
- Align content appropriately

## 🔧 Common Commands

### Fetch transcript only
```bash
node ytsum.js https://youtu.be/VIDEO_ID
```

### Fetch with summary prompt
```bash
node ytsum.js --summary https://youtu.be/VIDEO_ID
```

### Generate voice-over audio
```bash
node ytsum.js --voice https://youtu.be/VIDEO_ID
```

### Voice-over with custom settings
```bash
node ytsum.js --voice --voice-model fast --voice-speed 1.2 https://youtu.be/VIDEO_ID
```

### Using slash command
```
/ytsum https://youtu.be/VIDEO_ID
```

### Read existing transcript
```
Read the transcript at output/transcript-VIDEO_ID.txt and summarize it
```

## 📌 Important Notes for Gemini

1. **Always prefer local transcript files** over fetching new ones if they exist
2. **Save summaries** to `output/summary-{VIDEO_ID}.md`
3. **Handle errors gracefully** - if transcript fetch fails, explain why and suggest alternatives
4. **Extract VIDEO_ID** from various URL formats:
   - `https://www.youtube.com/watch?v=VIDEO_ID`
   - `https://youtu.be/VIDEO_ID`
   - `https://youtube.com/embed/VIDEO_ID`
   - `https://youtube.com/shorts/VIDEO_ID`
   - Just `VIDEO_ID` (11 characters)

5. **Video ID is always 11 characters** containing: `a-z`, `A-Z`, `0-9`, `-`, `_`

## 🛠 Troubleshooting

### "No transcript found"
- Video may not have captions
- Try different language: `node ytsum.js --lang es <url>`
- Some videos disable captions

### "Private video"
- Cannot access private videos
- Video must be public or unlisted

### "Age restricted"
- Age-restricted videos need authentication
- Not supported in this tool

### "Network error"
- Check internet connection
- Try again in a few moments

## 🏆 Best Practices

1. **Read the entire transcript** before summarizing
2. **Identify the main argument** or purpose of the video
3. **Note timestamps** of important moments while reading
4. **Extract verbatim quotes** rather than paraphrasing
5. **Create logical chapters** based on topic changes
6. **Highlight actionable content** prominently
7. **Format code properly** with language hints
8. **Make it scannable** — someone should get value in 30 seconds
