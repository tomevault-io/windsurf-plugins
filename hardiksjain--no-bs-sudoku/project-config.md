# no bs sudoku

a design-forward sudoku app built with flutter. dark, minimal, offline-first. no ads, no paywalls, no dark patterns.

## commands

```bash
fvm flutter pub get                                          # install dependencies
fvm dart run build_runner build --delete-conflicting-outputs  # generate drift code
fvm flutter run                                              # run the app
fvm flutter test                                             # run all tests
fvm flutter analyze                                          # lint check
```

always use `fvm` to run flutter/dart commands. after changing any drift table in `lib/core/storage/app_database.dart`, regenerate with `build_runner`.

## tech stack

- **flutter** (dart ^3.11.1) — single codebase for iOS + android
- **flutter_bloc** — state management via cubit pattern (not full bloc)
- **drift** + sqlite — type-safe local persistence (6 tables)
- **go_router** — declarative routing
- **fl_chart** — sparklines and bar charts
- **flutter_animate** — staggered animations
- **google_fonts** — DM Mono (numbers), Space Mono (UI text)
- **pure dart puzzle engine** — no external solver dependencies

## architecture

```
lib/
  engine/              pure dart puzzle engine (generation, solving, board model)
  core/
    intelligence/      quality score, velocity analysis, daily insights
    storage/           drift database + StorageService wrapper
    theme/             colors, typography, spacing, theme cubit
    routing/           go_router setup
  features/
    splash/            boot screen
    home/              daily card, stats strip, resume bar
    game/              game screen, cubit, grid, number pad
    complete/          solved screen with staggered animations
    stats/             performance dashboard (sparkline, heatmap)
    settings/          preferences, theme picker, data management
```

## patterns and conventions

- **state management:** one cubit per feature. immutable state objects with `copyWith()`. sealed classes for action unions (e.g. `GameAction`). use `buildWhen` to skip unnecessary rebuilds (especially timer ticks)
- **storage:** all DB access goes through `StorageService` — never access drift tables directly
- **naming:** PascalCase for classes/enums, camelCase for everything else, `_private` prefix for private members
- **linting:** flutter_lints defaults, no custom overrides
- **no backend:** fully offline. daily puzzles use seeded deterministic generation (same date = same puzzle globally)
- **generated code:** `app_database.g.dart` is generated by drift — don't edit manually

## design

- **palette:** background `#0A0A0A` (dark) / `#000000` (amoled), accent `#C8FF00` electric lime, surface `#111111`
- **typography:** DM Mono for all numbers, Space Mono for all UI text
- **style:** premium minimalist. lots of negative space. no splash/ripple effects. animations are fast and subtle, never flashy or decorative
- **copy voice:** lowercase, dry, calm. no exclamation points. ever.
  - say "Solved." not "Amazing!! You did it!"
  - say "7 days. keep going." not "You're on fire!"
  - say "something broke. try again." not "Oops! Something went wrong."

## testing

6 test suites covering the engine (generator, solver, board), intelligence (quality score, velocity), and widgets. run with `fvm flutter test`.
