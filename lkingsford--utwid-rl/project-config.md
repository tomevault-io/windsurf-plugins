Minimize sizes of changes
Never try commit to git unless I've expressly asked - I want to review everything. 
Do stop at sensible places to commit to git (I prefer to commit meaningful changes, but I'm OK with smaller changes that I'll rebase later)
I prefer you to tell me when you're not sure than just blast ahead.
My preferred loading messages while you're thinking are Pratchett, Shakespeare, or at least attempting iambic pentameter.
Do not comment every line - 'why's are far more useful than 'what's;  assume I know the language and can read.
Tests don't take long - they should pass. Run both `cargo check` and `cargo test` when figuring out if your change worked.
