# CLAUDE.md

## Project Mission

This repository is focused on **decompiling, understanding, and rebuilding Outpost (DOS/Windows era)**.

Primary goals:

- Recover readable/maintainable source from decompiler output.
- Reconstruct buildable C/Rust code paths.
- Preserve behavioral parity with the original game.

---

## Key Top-Level Directories

- `decompiled_c/` — main exported C/H source from Ghidra, compatibility headers, and split artifacts.
- `outpost_recompiled_c/` — active self-contained C/C++ restructuring workspace for rewiring and compile recovery.
- `reversing/` — reverse-engineering project assets, including the Ghidra project and Binary Ninja databases.
- `resources/` — original binaries and game asset files extracted from the original CD-ROM.
- `archive/` — historical and intermediary conversion artifacts.
- `scripts/` — automation for analysis, orchestration, and code splitting/indexing.

---

## `decompiled_c/` Structure (Where to Find Things)

- `decompiled_c/outpost.c`
  - Large monolithic Ghidra-exported C source.
- `decompiled_c/outpost.h`
  - Large monolithic exported header with structs/typedefs/prototypes.
- `decompiled_c/ghidra.h`
  - Compatibility macros/types for decompiler-generated code patterns.
- `decompiled_c/CMakeLists.txt`
  - Baseline CMake scaffold for attempting rebuild/compilation experiments.
- `decompiled_c/split_outpost/`
  - LLM-friendly split outputs and indexes generated by `scripts/split_ghidra_exports.py`.
  - Important files:
    - `MANIFEST.md` — first-pass split inventory.
    - `outpost_c/` and `outpost_h/` — size-based chunks.
    - `themed/` — thematic chunks and third-pass indexes:
      - `THEMED_MANIFEST.md`
      - `THEMED_INDEX.json` / `THEMED_INDEX.md`
      - `THEMED_FUNCTION_INDEX.json` / `THEMED_FUNCTION_INDEX.md`
      - `outpost_c/<theme>/...`
      - `outpost_h/<theme>/...`

## `outpost_recompiled_c/` Structure (Active Reconstruction Workspace)

- `outpost_recompiled_c/CMakeLists.txt`
  - Standalone build/config scaffold for recompiled C/C++ work.
- `outpost_recompiled_c/ghidra.h`
  - Local compatibility header copy for self-contained builds.
- `outpost_recompiled_c/themed/`
  - Working copy of themed split outputs and indexes.
- `outpost_recompiled_c/LLM_REWIRE_PLAN.md`
  - Stepwise task plan for scaffolding, moving files, include rewiring, placeholder dependency handling, and compile stabilization.

Primary objective in this folder:

- Resolve structural and include/link issues incrementally until code paths are compile-capable,
- track unresolved dependencies with placeholder files and explicit Ghidra follow-up notes.

### Active vs reference policy (required)

- Treat `decompiled_c/split_outpost/themed/` as an **immutable reference corpus**.
- Perform active rewiring, include fixes, and build-oriented edits in `outpost_recompiled_c/`.
- If a dependency is unknown, route through `include/core/missing_deps.h` and log it in
  `outpost_recompiled_c/docs/MISSING_DEPENDENCIES.md`.
- Resolve each placeholder via a **Ghidra research loop** (`reversing/ghidra_projects/` + MCP validation), then replace temporary declarations with verified ones.

## Reverse-Engineering Source of Truth

The Ghidra project is located under:

- `reversing/ghidra_projects/`

When uncertainty exists in decompiled output, **refer back to Ghidra** for validation. Prefer using the **Ghidra MCP tooling** to:

- inspect decompilation and disassembly at source,
- verify symbols/structures/callers,
- correct assumptions before refactoring or porting.

## Original Game Data

`resources/` includes:

- `resources/assets/binaries/` — original executable/binary material,
- `resources/assets/` subfolders — original asset files from the CD-ROM (audio, maps, images, music, UI, shaders, etc.),

Treat these as canonical reference data during rebuild verification.

## Recommended Workflow

1. Start with themed/function index files in `decompiled_c/split_outpost/themed/`.
2. Navigate to the relevant chunk file(s) instead of opening monolithic files.
3. For active rewiring/refactor work, operate in `outpost_recompiled_c/` and follow `outpost_recompiled_c/LLM_REWIRE_PLAN.md`.
4. Cross-check unclear logic in `reversing/ghidra_projects/` using Ghidra MCP or ReVa MCP.
5. Keep behavior-aligned notes when translating to cleaner implementations.

## Phase 2: Decompiler Artifact Resolution

Phase 2 of the rewire plan focuses on fixing Ghidra decompiler artifacts.
See `outpost_recompiled_c/LLM_REWIRE_PLAN.md` for the full task list.

### Key artifact types

| Artifact | Pattern | Fix |
| --- | --- | --- |
| Register args | `in_BX`, `in_DX`, `in_AX` | Promote to function parameters (verify via Ghidra/ReVa MCP) |
| Callee-saved regs | `unaff_SS`, `unaff_SI` | Remove segment refs, convert others to params/locals |
| Pointer construction | `CONCAT22(seg, off)` | Replace with flat pointer or integer expression |
| Win16 API calls | `SendMessage16(...)` | Route through `include/platform/win16_api.h` shim layer |
| Hex constants | `0x0111` | Replace with symbolic names (`WM_COMMAND`, resource IDs) |
