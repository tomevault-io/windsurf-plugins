## Project Overview

This is a research/creative project implementing **Request Confirmation Networks (ReCoN)** based on the paper `docs/CoCoNIPS_2015_paper_6.pdf` (included in the repo). The goal is to build a working implementation and dynamic visualization of a ReCoN, then demonstrate its use on an interesting representation problem.

This is a submission for the CIMC Research Engineer Creative/Technical Challenge (Option 5). The challenge values: problem scoping, design choices, clarity/creativity of solution, and clear documentation of process, limitations, and trade-offs.

## Key Files

- `docs/CoCoNIPS_2015_paper_6.pdf` — The ReCoN paper that defines the system to implement
- `docs/cimc-research-engineer-creative-technical-challenge.md` — Full challenge description with all five options
- `docs/cimc-show-my-thought-process.md` — Running notes on decision-making and thought process
- `docs/design.md` — Design document for the ReCoN Snake Mentor concept
- `docs/recon-overview.md` — Plain-language explanation of ReCoN

## Challenge Goals (from the prompt)

1. Build a working ReCoN implementation
2. Create a dynamic visualization of the network
3. Demonstrate ReCoN solving an interesting representation problem
4. Bonus: learning new representations, conversion into different formats
