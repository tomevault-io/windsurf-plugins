# Project Overview

This project is a Python-based "AI Debate Club" application with a graphical user interface built using `customtkinter`. The application facilitates a debate between two AI agents, a "PRO" and a "CON" debater, on a given topic. After the debate, a third AI agent, the "Judge," determines the winner.

The application is designed to be flexible and can be configured to use either a local language model via LM Studio or an online API (e.g., OpenAI's GPT models). Additionally, it includes a Text-to-Speech (TTS) feature to read the debate aloud, with support for both Microsoft Edge's TTS service and OpenAI's TTS API.

## UI Overview

The application's UI is organized into several frames:

*   **API Configuration:** Allows the user to switch between "Local LM Studio" and "Online API" modes.
    *   In "Online API" mode, the user can enter their API URL, API key, and model name.
    *   In "Local LM Studio" mode, the user can select the models for the topic selector/judge, PRO debater, and CON debater from a dropdown list of available models.
*   **Debate Configuration:** Allows the user to set the number of debate rounds and to manually enter a debate topic.
*   **TTS Configuration:** Allows the user to enable or disable TTS, to select the TTS engine ("Edge TTS" or "OpenAI TTS"), and to adjust the speech rate.
*   **Debate Transcript:** Displays the real-time transcript of the debate, including the final judgment.
*   **Controls:** Contains buttons to start and cancel the debate, a help button, and a status label.

## Key Features

*   **Graphical User Interface:** A user-friendly interface for configuring and running the debate.
*   **Dual AI Modes:**
    *   **Local Mode:** Connects to a local LM Studio server to use locally running language models.
    *   **Online Mode:** Connects to an online API (like OpenAI) for more powerful models.
*   **Automated Judge:** After the debate concludes, an AI judge analyzes the transcript and declares a winner with a justification.
*   **Text-to-Speech (TTS):**
    *   Supports both Edge TTS and OpenAI TTS with distinct voices for each debater and the judge.
    *   Allows for adjusting the speech rate.
    *   Generated audio files are automatically saved and organized into a `debates` directory, with subdirectories for each debate topic.
*   **Debate Configuration:**
    *   Set the number of rounds for the debate.
    *   Manually input a debate topic or have one generated by an AI.
*   **Transcript Management:**
    *   View the debate transcript in real-time.
    *   Copy the transcript to the clipboard.
    *   Save the transcript as a Markdown file.
*   **In-App Help:**
    *   A help button provides articles on how to set up LM Studio and obtain an OpenAI API key.
*   **Error Logging:** All errors are logged to an `error.log` file for easier debugging.
*   **Icon Buttons:** All buttons have icons to improve the user experience.
*   **Maximized Start:** The application starts in a maximized state for a better user experience.
*   **Share Debate:** The application can zip up the debate and upload it to a file sharing service, and the link is copied to the clipboard.

# Building and Running

This is a Python application and requires Python to be installed. The project dependencies are listed in the `requirements.txt` file.

## Running the Application

1.  **Install Dependencies:**
    ```bash
    pip install -r requirements.txt
    ```

2.  **Set up Environment Variables:**
    Create a `.env` file in the root of the project and add your OpenAI API key if you plan to use the online mode or OpenAI TTS:
    ```
    OPENAI_API_KEY="your-openai-api-key"
    ```

3.  **Run the application:**
    ```bash
    python app.py
    ```

# Development Conventions

*   **API Keys:** API keys are managed through a `.env` file. The application provides a way to save the API key from the UI.
*   **Modularity:** The application is structured within a single `app.py` file, with a `DebateApp` class encapsulating the application's logic and UI.
*   **Threading:** The application uses threading to run the debate logic, model fetching, and audio playback in the background to keep the UI responsive.
*   **Error Handling:** The application includes basic error handling for API requests and TTS generation.
*   **Code Style:** The code follows standard Python conventions.
*   **Audio Playback:** The application uses `pygame` to play the generated TTS audio files.
*   **File Organization:** Generated TTS audio files and debate transcripts are saved in a `debates` directory, with subdirectories for each debate topic.
