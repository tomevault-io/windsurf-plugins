## Project Overview

node-netmask is a Node.js library for parsing and manipulating IPv4 and IPv6 CIDR blocks. IPv4 supports various notations including dotted-quad, octal, hex, and shorthand forms. IPv6 supports full, compressed (`::`), and mixed IPv4-mapped forms.

## Commands

- **Build**: `npx tsc` (compiles TypeScript source to JS + declarations in `dist/`)
- **Test all**: `npm test` (compiles then runs mocha tests via ts-node)
- **Test single file**: `mocha --require ts-node/register tests/netmask.ts`

## Architecture

The source of truth is `lib/netmask.ts`. The `dist/` directory (JS + declarations) is generated by `tsc` and gitignored — it is only built at publish time.

Key internals:
- **IPv4**: `ip2long`/`long2ip`/`parseNum` for address parsing; `Netmask` class using 32-bit `number` math
- **IPv6**: `ip6bigint`/`bigint2ip6`/`parseIPv6Pure` for address parsing; `Netmask6` class using 128-bit `bigint` math
- IPv6 has no broadcast concept — `Netmask6.broadcast` is always `undefined`

Test files in `tests/`:
- `tests/netmask.ts` — IPv4 fixture table parsing, containment, forEach, hex/octal handling
- `tests/badnets.ts` — IPv4 invalid input error cases
- `tests/netmask6.ts` — IPv6 parsing, properties, containment, forEach, edge cases

## Security Notes

This library has had CVEs related to IP parsing (CVE-2021-29418, CVE-2021-28918). Octal/hex handling in `ip2long`/`parseNum` is security-sensitive — changes to IP parsing must be tested carefully against bypass vectors.
