> See `.claude/context.md` for workflow procedures, MCP usage, store patterns, and task templates.

## Project

**Svelte ToDo Kanban (ToDzz)** — Kanban task manager  
**Stack**: SvelteKit + Hasura GraphQL + PostgreSQL + Auth.js  
**Key features**: Drag-and-drop, rich text (Tiptap), file attachments, multi-user boards with roles, invitations, activity logs, GitHub integration, time tracking, dark mode, i18n, PWA

## Architecture

```
src/
├── routes/
│   ├── [lang]/[username]/[boardAlias]/[todoAlias]/  # Main app routes
│   ├── api/                                          # Upload, auth, GitHub webhooks
│   └── signin/
├── lib/
│   ├── components/
│   │   ├── todo/        # Todo detail components
│   │   ├── listBoard/   # Board & list components
│   │   ├── ui/          # shadcn-svelte shared components
│   │   └── ...          # auth, notifications, activity, editor, card, settings, github
│   ├── stores/          # State management (*.svelte.ts factory pattern)
│   ├── graphql/
│   │   ├── client.ts    # request() function
│   │   ├── documents.ts # ALL queries & mutations go here
│   │   └── generated/   # Auto-generated types (never edit)
│   ├── locales/         # i18n: en.json, et.json
│   └── schemas/         # Zod validation schemas
hasura/
├── migrations/          # DB migrations
├── metadata/            # Hasura schema, permissions, relationships
└── seeds/               # Test data
```

**Data flow**: Component → Store method → `request()` → Hasura → PostgreSQL → reactive `$state` update → re-render

**URLs**: `/[lang]/[username]/[boardAlias]/[todoAlias]` — aliases auto-generated by PostgreSQL triggers (lowercase-hyphenated, globally unique for boards, user-scoped for todos)

## Database Schema

**Core tables**:
- `users` — id, name, username, email, image
- `boards` — id, name, alias, user_id, sort_order, settings, github
- `lists` — id, name, board_id, sort_order
- `todos` — id, title, content, list_id, assigned_to, due_on, has_time, priority, completed_at, sort_order, min_hours, max_hours, actual_hours, github_issue_number, github_issue_id, github_synced_at
- `comments` — id, todo_id, user_id, content
- `uploads` — id, todo_id, url
- `labels` — id, user_id, name, color
- `todo_labels` — todo_id, label_id
- `board_members` — id, board_id, user_id, role (owner/editor/viewer)
- `board_invitations` — id, board_id, inviter_id, invitee_email, status
- `notifications` — id, user_id, type, todo_id, read_at
- `activity_logs` — id, user_id, todo_id, action, changes
- `logs` — id, user_id, level, component, message, metadata
- `notes` — id, user_id, title, content

**Tracker tables** (time tracking): tracker_sessions, tracker_apps, tracker_categories, tracker_keywords, tracker_category_apps, tracker_daily_stats, tracker_category_stats

**Permissions**: defined in `hasura/metadata/databases/default/tables/*.yaml` — users can only access their own boards or boards they're members of.

## Store Pattern (Svelte 5)

```typescript
import { browser } from '$app/environment';
import { request } from '$lib/graphql/client';

function createStore() {
  const state = $state({ items: [], loading: false, error: null });

  async function loadItems() {
    if (!browser) return;
    state.loading = true;
    state.error = null;
    try {
      const data = await request(GET_ITEMS, {});
      state.items = data.items || [];
    } catch (e) {
      state.error = e.message;
    } finally {
      state.loading = false;
    }
  }

  async function updateItem(id, updates) {
    const idx = state.items.findIndex(i => i.id === id);
    const original = { ...state.items[idx] };
    state.items[idx] = { ...original, ...updates }; // optimistic
    try {
      const data = await request(UPDATE_ITEM, { id, updates });
      state.items[idx] = data.update_items.returning[0];
      return { success: true, message: 'Updated' };
    } catch (e) {
      state.items[idx] = original; // rollback
      return { success: false, message: e.message };
    }
  }

  return {
    get items() { return state.items; },
    get loading() { return state.loading; },
    get error() { return state.error; },
    loadItems, updateItem
  };
}

export const myStore = createStore();
```

## Key Rules

- **Branch naming**: start with task file name — `git checkout -b 039-featureName`
- **Mutations**: always return `{ success: boolean, message: string, data? }`
- **Browser guard**: `if (!browser) return;` before any client-side API call
- **Optimistic updates**: always include rollback on error
- **Svelte 5 only**: use `$state`, `$derived`, `$effect`, `$props` — never legacy `$:` or `export let`
- **State**: single `$state({})` object per store, expose via getters only
- **Files**: 200–300 lines max per file; extract when larger
- **i18n**: `$t('key')` for all user-facing strings (locales in `src/lib/locales/`)
- **Logging**: `loggingStore.error/warn/info/debug('Component', 'msg', {data})` — view at `/[lang]/logs`
- **User feedback**: `displayMessage('msg', durationMs?, isSuccess?)` from `$lib/stores/errorSuccess.svelte`
- **No `prettier --write .`** — never format the entire codebase at once
