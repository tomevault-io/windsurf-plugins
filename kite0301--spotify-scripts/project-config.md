# Gemini-Spotify-Scripts Project

## Project Overview

This is a hobby development project by @Kite0301. It utilizes the Spotify Web API to create various scripts. The project is unique in its approach, using the Gemini CLI to generate the actual scripts based on requirements defined in Markdown files.

The main technologies used are:
- Spotify Web API
- Gemini CLI

The project structure is as follows:
- `docs/requirements`: Contains Markdown files that define the requirements for the scripts.
- `scripts`: This directory is initially empty and will be populated with scripts generated by the Gemini CLI.
- `.env`: This file should be created based on `.env.example` and contain the necessary Spotify API credentials.

## Building and Running

There are no explicit build commands. The project relies on the Gemini CLI to generate scripts.

To run the project, you would typically follow these steps:

1.  **Set up the environment:**
    *   Create a `.env` file in the root directory.
    *   Add the following environment variables to the `.env` file, replacing the placeholder values with your actual Spotify API credentials:
        ```
        SPOTIFY_CLIENT_ID=your_client_id_here
        SPOTIFY_CLIENT_SECRET=your_client_secret_here
        SPOTIFY_REDIRECT_URI=http://localhost:8888/callback
        ```

2.  **Generate scripts:**
    *   Use the Gemini CLI to read the requirement files in the `docs/requirements` directory.
    *   Instruct the Gemini CLI to generate the corresponding scripts in the `scripts` directory.

3.  **Execute scripts:**
    *   The generated scripts can be executed from the command line.

- **Script Generation:** The Gemini CLI is the primary tool for generating scripts based on the defined requirements.
- **Language:** The language for the generated scripts is not explicitly defined, but it is likely to be a common scripting language like Python or JavaScript, given the context of the Spotify Web API.
