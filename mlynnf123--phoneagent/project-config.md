## Project Overview

This is an AI-powered outbound calling assistant built with Node.js/Express.js. It processes leads from CSV files, makes automated calls via Twilio, uses Google Gemini for intelligent conversations, and integrates with Google Workspace (Calendar, Gmail, Sheets) and Notion for a complete calling workflow.

## Architecture

The codebase follows a modular service-oriented architecture:

- **handlers/** - Express route handlers and webhook processors
- **services/** - Business logic for external integrations (Twilio, Google, Notion, AI)
- **stores/** - Data persistence layer using Redis for conversations and job tracking
- **utils/** - Shared utilities like logging configuration

Key architectural patterns:
- Asynchronous job processing for CSV lead imports
- Redis-based session management for call conversations
- Webhook-driven call flow handling via Twilio
- Service layer abstraction for all external APIs

## Development Commands

```bash
# Install dependencies
npm install

# Start development server with hot-reload
npm run dev

# Start production server
npm start

# Run with Docker Compose (includes Redis and ngrok)
docker-compose up

# Run Redis locally (required for development)
docker run -d -p 6379:6379 redis:alpine
```

## External Service Dependencies

This project requires several external services to be configured via environment variables:

1. **Twilio** - For outbound calling capabilities
2. **Google Cloud** - Text-to-Speech, Speech-to-Text, and service account credentials
3. **Google Workspace APIs** - Calendar, Gmail, and Sheets access
4. **Google Gemini** - For AI conversation handling
5. **Notion API** - For knowledge base queries
6. **Redis** - For conversation and job state management

## Key Integration Points

### Twilio Webhooks
- `/webhook/call-handler` - Handles call events (answer, speech, gather)
- `/webhook/call-status` - Receives call status updates
- Requires public URL exposure (ngrok in development)

### Google Services Integration
- Uses service account authentication via `google-credentials.json`
- Calendar API for availability checking and appointment booking
- Gmail API for sending follow-up emails
- Sheets API for logging call activity

### Conversation Flow
1. Lead upload triggers async processing via `leadProcessor.js`
2. Twilio initiates calls using `twilioService.js`
3. Call events handled by `callHandler.js`
4. AI responses generated by `aiService.js` using Gemini
5. Context enriched by `notionService.js` queries
6. Results logged to Google Sheets via `googleService.js`

## Important Considerations

### No Test Framework
Currently no testing infrastructure exists. When implementing tests:
- Consider Jest or Mocha for unit testing
- Mock external service calls (Twilio, Google APIs, etc.)
- Test webhook handlers with simulated Twilio requests

### Environment Configuration
All sensitive data must be in `.env` file (already gitignored). Required variables:
- Twilio credentials and phone number
- Google service account path
- API keys for Gemini and Notion
- Redis connection URL
- Webhook base URL (ngrok URL in development)

### File Handling
- CSV uploads are temporarily stored in `uploads/` directory
- Files are deleted after processing
- Both `uploads/` and `logs/` directories are gitignored

### Error Handling
- Winston logger configured in `utils/logger.js`
- Service methods include try-catch blocks
- Failed calls logged to Google Sheets with error details

## Common Development Tasks

### Adding New Call Features
1. Modify conversation prompts in `aiService.js`
2. Update call flow logic in `callHandler.js`
3. Test with Twilio webhook simulator or actual calls

### Modifying Lead Processing
1. Update CSV parsing logic in `server.js`
2. Modify lead schema in `leadProcessor.js`
3. Ensure job status updates in `jobStore.js`

### Debugging Webhooks
1. Use ngrok web interface at `http://localhost:4040`
2. Check Winston logs for detailed request/response data
3. Verify webhook URL in Twilio console matches ngrok URL

### Working with Redis
- Conversation state: `conversation:{callSid}` keys
- Job tracking: `job:{jobId}` keys
- Default TTL: 3600 seconds (1 hour) for conversations
