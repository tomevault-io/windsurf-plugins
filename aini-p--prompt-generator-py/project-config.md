# プロジェクト概要

このプロジェクトは、Stable Diffusionのような画像生成モデル向けのプロンプトを生成するためのデスクトップアプリケーションです。Pythonと、GUIフレームワークであるPySide6で構築されています。このアプリケーションを使用することで、ユーザーはキャラクター、衣装、シーンなどの様々なプロンプト構成要素のデータベースを管理し、それらを組み合わせて複雑なプロンプトを生成することができます。

アプリケーションのメインウィンドウは、いくつかのパネルで構成されています。
-   **プロンプト生成 (Prompt Generation):** このパネルでは、シーンを選択し、そのシーンとアクターの割り当てに基づいてプロンプトを生成することができます。
-   **バッチ（シーケンス）(Batch (Sequence)):** このパネルでは、プロンプトをバッチで生成するためのシーンのシーケンスを作成・管理できます。
-   **ライブラリ (Library):** このパネルでは、プロンプト構成要素のデータベースを管理できます。
-   **データ管理 (Data Management):** このパネルでは、データの保存、エクスポート、インポート、同期などのオプションが提供されます。

アプリケーションは、すべてのデータを保存するためにSQLiteデータベースを使用します。データベースのスキーマは`src/models.py`で定義されています。

プロンプト生成のコアロジックは`src/prompt_generator.py`にあります。このモジュールは、シーン、アクターの割り当て、外見の上書きなどの様々な入力を受け取り、`GeneratedPrompt`オブジェクトのリストを生成します。

また、このプロジェクトには`StableDiffusionClient`というサブモジュールが含まれており、これはStable Diffusion APIと対話するためのクライアントのようです。`generation_worker.py`ファイルがバックエンドとタスクの投入を管理します。

## ビルドと実行

アプリケーションを実行するには、Pythonがインストールされている必要があります。その後、`run_app.bat`スクリプトを実行します。このスクリプトは以下の処理を行います。
1.  `.venv`ディレクトリにPythonの仮想環境が存在しない場合は作成します。
2.  仮想環境をアクティベートします。
3.  `requirements.txt`から必要な依存関係をインストールします。
4.  `main.py`を実行してアプリケーションを起動します。

初回実行時、アプリケーションはデータベースファイルを要求します。新しいファイルを作成するか、既存のファイルを選択することができます。

## 開発規約

このプロジェクトは、様々な機能が異なるファイルやディレクトリに分割された、モジュール式の構造に従っています。
-   `src/models.py`: アプリケーションのデータモデルを定義します。
-   `src/database.py`: データベースとの対話を処理します。
-   `src/main_window.py`: メインのアプリケーションウィンドウです。
-   `src/panels/`: メインウィンドウで使用される様々なパネルが含まれています。
-   `src/widgets/`: アプリケーションで使用される様々なカスタムウィジェットが含まれています。
-   `src/handlers/`: データハンドラが含まれています。
-   `src/prompt_generator.py`: プロンプト生成のコアロジックが含まれています。
-   `src/generation_worker.py`: バックエンドとタスクの投入を管理します。

コードでは型ヒントが多用されており、これによりコードの可読性と保守性が向上しています。また、異なるコンポーネント間の通信には、PySideのシグナルとスロットのメカニズムが使用されています。
