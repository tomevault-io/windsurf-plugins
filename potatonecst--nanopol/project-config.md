# Gemini Code Assist Guidelines

このドキュメントは、このプロジェクトにおけるAIアシスタント（Gemini）の振る舞いに関するガイドラインです。
## Coding Standards

## 0. 重要事項 (Critical Instructions)
### TypeScript / React

**【最優先】使用言語について:**
ユーザーとの対話、変更箇所の説明、作業計画の提示は当然として、**ツールを実行する直前の行動予定の説明（いわゆる "I will..." から始まる文章）も、必ず日本語で記述してください。**
英語での出力は、コード内のコメントや、英語での出力が自然な識別子などを除き、原則禁止とします。

## 1. プロジェクト概要

本プロジェクトは、ナノ粒子の偏光散乱光を測定・解析するためのデスクトップアプリケーション「NanoPol Controller」の開発です。
詳細については `spec/` ディレクトリ内の仕様書を参照してください。

*   **目的:** ナノ粒子（~150nm）からの散乱光強度を、1/4波長板の回転角度ごとに精密測定し、実験の完全なトレーサビリティを確保する。
*   **主要ハードウェア:** 回転ステージ (OptoSigma GSC-01), カメラ (Thorlabs DCC1645C)

## 2. ドキュメント作成・更新ルール

コードに変更を加えた場合、以下の `docs/` または `spec/` 内の関連ファイルもセットで更新することを義務とします。

*   `docs/00_architecture.md`: 構成変更時
*   `docs/01_backend_devices.md`: `backend/devices/` 変更時
*   `docs/02_backend_server.md`: `backend/main.py` 等変更時
*   `docs/03_frontend_client.md`: フロントエンド変更時

更新の際は、単なるコードのコピペではなく、以下の点を解説として追記・修正してください：
- **変更の意図**
- **新しいコンポーネント・関数の仕様（Props、引数、戻り値）**
- **使用上の注意点**

さらに、ユーザーの学習を支援するため、**コードの動作原理、使用している技術やパターンの背景、なぜその実装を選択したか**についても、可能な限り詳細に解説を記述してください。
（例：Reactのライフサイクル（マウント/アンマウント/再描画）の挙動、`useEffect`の依存配列の仕組み、Zustandによる状態管理、TauriのIPC通信、Zodによるバリデーションの仕組み、ハードウェア制約とソフトウェア制御の関係など）

**【重要】解説の粒度について:**
ユーザーは自身でコードを記述していないため、実装の詳細がブラックボックス化することを避ける必要があります。
「どの行が何をしているか」「なぜこのフックが必要なのか」「React/Tauri特有の作法はどれか」といった、コードを読み解くための補助線を引くような解説を心がけてください。

## 3. 基本的な開発方針

### 仕様書ファースト (Spec First)
新規機能の追加や既存機能の大幅な変更を行う際は、まず `spec/` ディレクトリ内の関連する仕様書を更新・修正してください。仕様書を基に議論や合意形成を行います。

### 仕様書と実装の同期 (Sync Spec & Code)
**重要:** コードを変更・実装する際は、必ず `spec/` (仕様書) および `docs/` (技術ドキュメント) との整合性を維持してください。

1.  **仕様変更時:** まず `spec/` を更新し、その要件に基づいてコードを実装してください。
2.  **実装変更時:** リファクタリングや詳細設計の変更を行った場合、即座に `docs/` 内の該当ファイルを更新してください。
3.  **乖離の禁止:** 仕様書、実装コード、技術ドキュメントの3者が常に一致している状態を保ってください。「コードだけ直してドキュメントは後で」は禁止です。

## 4. 技術スタックと規約
- **UI:** shadcn/ui および Tailwind CSS を採用しています。新規UIコンポーネントを追加する際は、既存のスタイルと設計思想に従ってください。

## 5. Geminiへの指示

- **事前確認:** コードを変更する前に、必ずプロジェクト内の関連ドキュメントや既存コードを確認し、全体像を把握してください。
- **UI実装:** 既存のコンポーネント (`src/components/ui/`) やレイアウト (`src/App.tsx` など) のスタイルを厳密に踏襲し、統一感を維持してください。
- **ドキュメントの整合性:** 機能の追加や修正が完了した際は、その内容がドキュメントに正しく反映されているかを確認し、必要であれば更新してください。
- **コードコメント:** 処理の意図や複雑なロジックについては、詳細かつ分かりやすいコメントを記述してください。

### 詳細な日本語コメントの徹底
- ユーザーは実装の詳細を把握していないため、コード自体が解説書を兼ねるレベルで詳細な日本語コメントを記述してください。
- 変数の意味、関数の目的、複雑なロジックの解説、Reactフック（`useState`, `useEffect`, `useRef` 等）の挙動など、冗長と思われる程度に丁寧に記述することを求めます。
- 特に、**TauriによるRust/Pythonバックエンドとの連携部分**や、**非同期処理（async/await）**については、背景知識がなくても理解できるように補足してください。

### TSX/JSXへのコメント記述
- **UI構造の解説**: 関数ロジックだけでなく、TSX/JSXのタグ部分にもコメントを挿入してください。
- **記述内容**: レイアウトの意図（Flex/Gridの使い分けなど）、各セクションの役割、コンポーネントの責任範囲、特定のインタラクション（クリック時の挙動など）について説明してください。
- **記述例**: `{/* メインレイアウト: サイドバーとコンテンツエリアを横並びに配置 */}`

**学習者への配慮:**
開発者はプログラミング学習中であることを念頭に置き、「どの関数・メソッドを呼び出しているか」「それによってどのような効果（再描画、非同期実行、デバイス制御など）が得られるか」といった、実装の仕組みレベルの解説もコメントに含めてください。
