# DRP Website UI Refactor Rules (Updated)

## ✅ Full Layout and Component Redesign Allowed

All previous restrictions that prevented layout changes, component replacement, or page restructuring are now removed.

The system is explicitly allowed to redesign and improve ALL pages of the website.

This includes permission to:

- change layouts across any page
- replace existing components with better ones
- introduce new reusable UI sections
- improve page hierarchy and visual flow
- standardize styling across all pages
- refactor shared layout wrappers and shells
- modernize outdated or flat sections
- unify the visual language across the full site

## 🎯 Design Objective

The DRP website should become a:

- premium
- futuristic
- cinematic
- blockchain-native
- AI-enhanced
- investor-ready
- human-centered

digital product experience.

## 🧩 Approved Design Enhancements

The system is encouraged to apply:

- modern page heroes
- premium feature grids
- bento layouts
- glassmorphism cards
- timeline sections
- ecosystem diagrams
- tabs and accordions
- CTA banners
- hover interactions
- scroll reveal motion
- animated gradients
- strong visual hierarchy

## 🌌 Site-Wide Consistency Requirement

All pages must feel like part of the same futuristic DRP design system.

No page should feel:
- unfinished
- plain
- outdated
- disconnected from the homepage
- visually weaker than the homepage

## ⚙️ Technical Guidance

When redesigning:
- preserve routing and functionality
- preserve content meaning
- improve maintainability
- favor reusable components over duplicated styling
- improve responsiveness and mobile UX
- refactor shared structures where needed

## 🚫 Avoid

Avoid:
- generic crypto styling
- cluttered sections
- weak spacing
- over-animation
- inconsistent UI patterns
- plain secondary pages

## 🧠 Core Principle

Every design change must improve at least one of the following:

- clarity
- trust
- elegance
- consistency
- usability
- futuristic feel
- emotional impact
