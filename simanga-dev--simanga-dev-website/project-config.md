# Project Overview

This is a personal website template built with [Next.js](https://nextjs.org/), [React](https://react.dev/), [Tailwind CSS](https://tailwindcss.com/), and [Motion](https://motion.dev/). It is designed for developers, designers, and founders who want to create a minimal and animated personal website. The template includes a blog that supports [MDX](https://mdxjs.com/).

## Building and Running

### Prerequisites

- [Node.js](https://nodejs.org/) (v20 or higher)
- [pnpm](https://pnpm.io/) (or `npm` / `yarn`)

### Installation

1.  Clone the repository:
    ```bash
    git clone https://github.com/ibelick/nim.git
    ```
2.  Navigate to the project directory:
    ```bash
    cd nim
    ```
3.  Install the dependencies:
    ```bash
    pnpm install
    ```

### Running the Development Server

To start the development server, run the following command:

```bash
pnpm run dev
```

This will start the development server on `http://localhost:3000`.

### Building for Production

To build the application for production, run the following command:

```bash
pnpm run build
```

This will create an optimized build of the application in the `.next` directory.

### Running in Production

To start a production server, run the following command:

```bash
pnpm run start
```

This will start a production server on `http://localhost:3000`.

## Development Conventions

### Linting

This project uses [ESLint](https://eslint.org/) for linting. To lint the code, run the following command:

```bash
pnpm run lint
```

### Code Formatting

This project uses [Prettier](https://prettier.io/) for code formatting. It is recommended to set up your editor to format the code on save.

### File Structure

The main files and directories of the project are:

-   `app/`: This directory contains the main application code, including the pages, layouts, and components.
-   `public/`: This directory contains the static assets of the application, such as images and fonts.
-   `components/`: This directory contains the reusable components of the application.
-   `lib/`: This directory contains the utility functions and constants of the application.
-   `hooks/`: This directory contains the custom hooks of the application.
-   `blog/`: This directory contains the blog posts in MDX format.
