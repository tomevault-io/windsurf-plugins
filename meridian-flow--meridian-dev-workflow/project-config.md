# meridian-dev-workflow

Agent profiles and skills for the dev orchestration workflow.

## Releasing

Use `meridian mars version` for releases:

```bash
# Patch release (0.0.29 → 0.0.30)
meridian mars version patch --push

# Minor release (0.0.30 → 0.1.0)  
meridian mars version minor --push

# Explicit version
meridian mars version 1.0.0 --push
```

This:
1. Bumps version in `mars.toml`
2. Commits the change
3. Tags the commit
4. Pushes branch and tag to origin

### Changelog

Update `CHANGELOG.md` before releasing:
1. Move `[Unreleased]` content to new version section
2. Add date: `[X.Y.Z] - YYYY-MM-DD`
3. Use caveman style — terse, fragment-friendly
4. Commit changelog, then run `meridian mars version`

### After Release

Downstream repos (like `meridian-cli`) pick up new versions via:
```bash
meridian mars add meridian-dev-workflow@vX.Y.Z
meridian mars sync
```

## Structure

```
agents/           # Agent profiles (.md with YAML frontmatter)
skills/           # Skill definitions (SKILL.md + resources/)
mars.toml         # Package manifest
mars.lock         # Locked dependencies
.mars/            # Generated — do not edit directly
```

## Editing

Edit source files in `agents/` and `skills/`. Never edit `.mars/` — it's generated by `meridian mars sync`.

After editing, test locally by running `meridian mars sync` in a downstream repo that imports this package.
