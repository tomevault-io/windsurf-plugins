# AGENTS Guide

This file defines expected behavior for AI and human contributors working on AI Police.

## Operating Principle

AI Police is a public-interest authority framework, not an extra-legal actor.

All contributions must:

- strengthen lawful governance,
- preserve transparency and accountability,
- remain adaptable for jurisdiction-specific legal requirements.

## Required Checks Per Change

- Is the contribution lawful by design and jurisdiction-aware?
- Does it preserve human oversight?
- Does it document audit and appeal implications?
- Does it avoid hard-coding political or private suppression logic?
- Are trust and safety docs updated consistently?

## Role Boundaries

- Guidance Agent: drafts policy logic, moderation flows, and escalation guidance.
- Compliance Agent: verifies lawful scope, privacy posture, and safety boundaries.
- Implementation Agent: adds technical scaffolding with auditability and rollback paths.

## Release Quality Gate

No change should be considered release-ready unless it includes:

- scope statement,
- risk notes,
- affected docs updates,
- verification notes.
