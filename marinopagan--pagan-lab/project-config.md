## What this is

A research lab website for the Pagan Lab (Simons Initiative for the Developing Brain, University of Edinburgh) built on the [Lab Website Template](https://greene-lab.gitbook.io/lab-website-template-docs) by Greene Lab. It uses Jekyll (Ruby) for static site generation and Python for automated citation management.

Deployed at: www.marinopagan.com/pagan_lab

## Development

**Docker (recommended):**
```bash
./.docker/run.sh
```
This builds the container and serves the site with live reload.

**Without Docker:**
```bash
# Generate citations first
python3 _cite/cite.py

# Serve with live reload
bundle exec jekyll serve --livereload --force_polling
```

**Production build:**
```bash
JEKYLL_ENV=production bundle exec jekyll build --baseurl "/"
```

**Citation generation only:**
```bash
python3 _cite/cite.py
```

## Architecture

### Citation pipeline

1. `_data/sources.yaml` — manually defined publication sources (DOIs, PubMed IDs, ORCID IDs, etc.)
2. `_data/orcid.yaml` — ORCID API configuration for auto-fetching publications
3. `_cite/cite.py` runs plugins in sequence: `google-scholar → pubmed → orcid → sources`
4. Plugins normalize entries to a common format; Manubot generates full citation metadata from identifiers
5. Output is written to `_data/citations.yaml` — **do not edit this file manually**

Citations are refreshed automatically via GitHub Actions on a weekly schedule (`on-schedule.yaml`) and on push (`on-push.yml`).

### Jekyll structure

- `_layouts/` — page templates (`default`, `member`, `post`)
- `_includes/` — reusable HTML components (citation cards, header, footer, buttons, etc.)
- `_styles/` — SCSS stylesheets (compiled by Jekyll)
- `_scripts/` — vanilla JS for search, dark mode, tooltips
- `_members/` — lab member profile pages (Markdown with front matter)
- `_posts/` — blog posts
- `_plugins/` — custom Ruby helpers (array, hash, file, regex utilities)

### Content data

All structured content lives in `_data/`:
- `citations.yaml` — auto-generated; do not edit
- `sources.yaml` — manually curated publication sources
- `projects.yaml` — research project descriptions
- `types.yaml` — publication type labels/icons

### Deployment

GitHub Actions workflows in `.github/workflows/` handle everything:
- `build-site.yaml` — builds and deploys to GitHub Pages on push to `main`
- `update-citations.yaml` — re-runs citation pipeline and commits updated `citations.yaml`
- `on-schedule.yaml` — weekly trigger for citation updates
