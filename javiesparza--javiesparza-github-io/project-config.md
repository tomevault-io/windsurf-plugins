## Development Guidelines

### General Principles
- Keep the site lightweight and fast-loading
- Ensure mobile responsiveness for all pages
- Maintain clean, semantic HTML structure
- Follow web accessibility best practices (WCAG guidelines)

### Code Style
- Use consistent indentation (2 spaces for HTML/CSS/JS)
- Write semantic HTML5 markup
- Prefer CSS classes over inline styles
- Use modern CSS features (flexbox, grid) for layouts
- Keep JavaScript vanilla unless there's a strong reason for dependencies

### File Organization
- Keep static assets organized in appropriate directories (css/, js/, images/, etc.)
- Use descriptive filenames with lowercase and hyphens (e.g., `about-me.html`)
- Maintain a logical folder structure as the site grows

### Content Guidelines
- Ensure all content is professional and appropriate for a portfolio site
- Optimize images for web (compress, use appropriate formats)
- Test all external links regularly
- Keep dependencies minimal

### Testing
- Test in multiple browsers (Chrome, Firefox, Safari, Edge)
- Verify mobile responsiveness at different screen sizes
- Check for broken links before deploying
- Validate HTML/CSS using W3C validators

### Deployment
- All changes pushed to the main branch are automatically deployed to GitHub Pages
- Preview changes locally before committing
- Keep commit messages clear and descriptive
