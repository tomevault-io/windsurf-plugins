# Skills Heroes - Web Developer 2024-2025 (Preliminary Round)

This document summarizes the key information from the competition briefing documents.

## 1. General Information

- **Competition:** Skills Heroes - Web Developer 2024-2025
- **Level:** MBO, Preliminary Round
- **Crebo:** 25604
- **Task:** Develop a portfolio website for an architectural firm.
- **Format:** Individual competition
- **Duration:** 180 minutes (3 hours)

## 2. The Assignment

You are a freelance web developer tasked with creating a portfolio website for the architectural firm **"LivingShapes"**.

- You are responsible for the entire design and implementation of the website.
- You must use the provided media assets and the `.sql` database export.
- You can decide how to get data from the back-end to the front-end.
- No specific design software (like Figma or Sketch) is required.

## 3. Technical Stack & Skills

You are expected to be proficient in the following technologies:

- **HTML5:**
    - Use semantic elements (`<main>`, `<section>`, `<header>`, `<footer>`, etc.).
    - Ensure your code is valid and scores well on performance, accessibility, and SEO (check with Lighthouse).
- **CSS:**
    - Use modern layout techniques like **Grid** and **Flexbox**.
    - The website must be **responsive** and work well on mobile devices.
- **JavaScript:**
    - Solid understanding of the basics (or TypeScript).
    - Know how to handle arrays and send `FormData` to a back-end.
    - Knowledge of a JS framework (React, Vue, Angular) is a plus.
- **Back-end:**
    - **PHP** is the recommended back-end language.
    - You are allowed to use **Node.js** if you are more comfortable with it.
    - Basic OOP knowledge is a plus.
- **SQL:**
    - Write basic queries (`INSERT`, `UPDATE`, `DELETE`, `LEFT JOIN`).
    - Be able to manage a database (e.g., via phpMyAdmin).
    - Connect the database to your back-end to retrieve and store data.
- **Git:**
    - You should be able to place your project in a Git repository.

## 4. Rules & Regulations

- **Internet access is allowed** for this round.
- **AI tools are strictly forbidden.** This includes ChatGPT, Bard, and any AI plugins in your IDE.
- **Communication tools are forbidden.** This includes social media and instant messaging applications.
- You may **not** keep a copy of the code after the competition.
- You may **not** publish the code to a public repository like GitHub.
- You must use the provided media assets.
- You can use online resources for reference.

**Allowed personal items:**
- Your own mouse and keyboard.
- A wired headset for listening to music (e.g., Spotify, YouTube).

## 5. Deliverables & Assessment

- **Deliverable:** A `.zip` file containing your project, including a `README.md` file that explains how to run it. Alternatively, you can leave the project on the computer for your instructor to assess.
- **Assessment Criteria:**
    - **Programming and Skills (21 points):** General coding quality and application of skills.
    - **Final Product Requirements (79 points):** Correct use of the database, design quality, and user-friendliness.
    - **Total:** 100 points.
