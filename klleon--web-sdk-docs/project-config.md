This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Klleon Chat SDK 공식 문서 사이트. Docusaurus 3.9.2 + React 19 + TypeScript 기반의 정적 다국어(ko/en/ja) 문서 사이트이다.

- 배포 URL: https://docs.klleon.io
- 기본 로케일: `ko` (한국어)
- SDK 현재 버전: 1.3.0
- 패키지 매니저: **npm** (yarn 사용 금지, `yarn.lock` 삭제됨)
- Node.js: **>= 20.0** 필수 (`engines` 필드로 강제)

## Commands

```bash
# 개발 서버 (기본 로케일: ko)
npm run start

# 특정 로케일로 개발 서버 실행
npm run start:en
npm run start:ko
npm run start:ja

# 프로덕션 빌드 (모든 로케일)
npm run build

# 빌드 결과물 로컬 미리보기
npm run serve

# 타입 체크
npm run typecheck

# 번역 파일 생성/업데이트
npm run write-translations:ko
npm run write-translations:en
npm run write-translations:ja

# Docusaurus 캐시 초기화
npm run clear
```

## Architecture

### 콘텐츠 구조

`docs/` 디렉토리의 MDX 파일이 문서 콘텐츠이며 디렉토리 숫자 접두사로 사이드바 순서를 제어한다. `sidebars.ts`는 `autogenerated` 방식으로 디렉토리 구조를 그대로 사이드바로 매핑한다.

```
docs/
├── intro.mdx              # 시작하기
├── 02-setup/              # 설치·초기화
├── 03-usage/              # API 사용법
├── 04-guides/             # 가이드 (TypeScript 지원 등)
├── 05-examples/           # Vanilla JS / React 예제
└── 06-changelog/          # 버전별 변경 이력 (v1.0.0 ~ v1.3.0)
```

### 다국어(i18n) 처리

- **문서 콘텐츠**: `docs/`의 MDX 파일이 소스(한국어)이며, `i18n/en/`, `i18n/ja/` 디렉토리에 각 로케일 번역 파일이 위치
- **UI 문자열**: `i18n/{ko,en,ja}/code.json`에 번역 키-값 저장
- **데이터 테이블**: `src/data/setup.js`, `src/data/usage.js`에서 locale별 파라미터 설명을 객체로 관리

### i18n 동기화 체크리스트

`docs/` 소스(한국어) 수정 시 아래 체크리스트를 반드시 확인한다:

- [ ] `i18n/en/docusaurus-plugin-content-docs/current/` 영어 번역 파일 동기화
- [ ] `i18n/ja/docusaurus-plugin-content-docs/current/` 일본어 번역 파일 동기화
- [ ] `src/data/setup.js` — `ko`, `en`, `ja` 3개 로케일 데이터 동기화
- [ ] `src/data/usage.js` — `ko`, `en`, `ja` 3개 로케일 데이터 동기화
- [ ] Mermaid 다이어그램 — 소스와 번역 파일의 flowchart/sequenceDiagram 구조 일치
- [ ] Admonition 문법 — `:::tip`, `:::caution`, `:::info` 등이 `\:::` 이스케이프 없이 정상 표기
- [ ] `npm run build` 후 `build/en/llms-full.txt`에 신규/수정 콘텐츠 포함 확인

### 홈페이지 구조

`src/pages/index.tsx`는 3섹션(Hero → Feature → CTA)을 조합하는 얇은 래퍼이다. 각 섹션은 독립 컴포넌트로 분리되어 있다.

```
src/components/home/
├── HeroSection.tsx + .module.css      — Hero 래퍼 (HeroInfo + SdkDemo 2열 레이아웃)
├── HeroInfo.tsx + .module.css         — Badge, 제목, 설명, CTA 버튼
├── SdkDemo.tsx + .module.css          — 라이브 SDK 데모 (useKlleonSdk 훅 사용)
├── FeatureSection.tsx + .module.css   — 3열 Feature 카드 그리드
├── FeatureCard.tsx + .module.css      — 개별 카드 (icon + title + desc)
├── CtaSection.tsx + .module.css       — 하단 CTA
└── icons/
    ├── ClockIcon.tsx                  — 타이머 SVG
    └── RefreshIcon.tsx                — 재연결 SVG
```

- **반응형**: 데스크톱(>1400px) 2열, 태블릿(768~1400px) 세로 스택, 모바일(<768px) SDK 데모 숨김
- **SDK 데모**: 60초 타이머 아바타 인터랙션. 타이머 만료 시 오버레이 + reconnect 버튼

### 공유 유틸리티

- `src/utils/locale.ts` — `getLocaleCode(locale)`: 로케일 → SDK 음성 코드 변환, `getDocsLink(locale)`: docs 링크 생성
- `src/utils/format.ts` — `formatTime(seconds)`: MM:SS 포맷

### 주요 컴포넌트

- `src/pages/index.tsx` — 랜딩 페이지 (HeroSection + FeatureSection + CtaSection 조합, ~25줄)
- `src/components/home/` — 홈페이지 섹션 컴포넌트 (위 구조 참조)
- `src/components/example/` — SDK 사용 예시 컴포넌트 (SetUp, BaseReact, CustomReact)
- `src/components/table/Table.tsx` — 제네릭 테이블 컴포넌트 (파라미터 문서용)
- `src/hooks/customHooks.ts` — `useTimer` 커스텀 훅
- `src/hooks/useKlleonSdk.ts` — SDK init/destroy/timer를 캡슐화한 커스텀 훅 (홈 데모용)
- `src/types/global.d.ts` — SDK 전역 타입 정의 (`KlleonChat`, `InitOption`, `ChatData`, `AvatarProps`, `ChatProps` 등)

### SDK 연동

SDK는 `docusaurus.config.ts`의 `scripts` 필드에서 UMD 번들(`https://web.sdk.klleon.io/1.3.0/klleon-chat.umd.js`)로 로드된다. `window.KlleonChat`으로 전역 접근하며 커스텀 웹 컴포넌트(`<avatar-container>`, `<chat-container>`)를 사용한다.

### 환경변수

`.env.development`와 `.env.production`에서 `REACT_APP_SDK_KEY`, `REACT_APP_AVATAR_ID`를 관리. `dotenv-cli`로 빌드/개발 시 주입한다. `docusaurus.config.ts`의 `customFields`에 fallback 값이 있어 `.env` 없이도 기본 동작한다.

## Design Conventions

### 브랜드 컬러

Primary 컬러는 `#3579CC` (파란색 계열)이며, Docusaurus Infima의 `--ifm-color-primary` 변수로 정의된다. 모든 파생 색상(dark, darker, light 등)도 파란색 계열로 통일되어 있다.

### CSS 변수 체계

`src/css/custom.css`의 `:root`에 `--klleon-color-*` 네임스페이스로 디자인 토큰을 정의한다. **컴포넌트나 페이지에서 색상을 사용할 때 반드시 CSS 변수를 참조해야 하며, 하드코딩된 hex/rgb 값을 직접 사용하지 않는다.**

| 카테고리 | 변수 접두사 | 용도 |
|---------|-----------|------|
| Accent | `--klleon-color-accent` | 버튼, 인터랙티브 요소 (`--ifm-color-primary` 연동) |
| Text | `--klleon-color-text-*` | heading, body, muted, inverse 텍스트 |
| Surface | `--klleon-color-surface-*` | primary(배경), secondary(그레이 배경), overlay |
| Border | `--klleon-color-border` | 입력 필드, 컨테이너 테두리 |
| CTA | `--klleon-color-cta-*` | Call-to-Action 버튼 (bg, hover) |
| Badge | `--klleon-color-badge` | 배지, 라벨 |
| Icon | `--klleon-color-icon-*` | 아이콘 원형 배경, 디테일, 액션 |
| Shadow | `--klleon-shadow-card` | 카드 그림자 |

### 새 색상 추가 시

1. `src/css/custom.css`의 `:root` 블록에 `--klleon-color-{카테고리}-{역할}` 형태로 변수를 추가한다.
2. 컴포넌트에서는 `var(--klleon-color-...)` 형태로만 참조한다.
3. Mermaid 다이어그램 등 MDX 인라인 스타일에서도 동일한 hex 값을 사용할 경우, 해당 값을 CSS 변수 정의와 동기화한다.

### CI/CD

- **Context7 문서 리프레시**: `.github/workflows/context7-refresh.yml` — `docs/`, `i18n/`, `src/data/` 경로에 변경이 push되면 Context7에 등록된 LLM 문서를 자동 리프레시한다. `workflow_dispatch`로 수동 실행도 가능하다.
