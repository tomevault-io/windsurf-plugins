# 项目整体要求

## 整体要求
* 禁止使用worktree
* 本项目中所有需要显示时间或格式化时间的地方，都使用本地时间（用户浏览器的时间）。不要写死任何固定时区，比如中国时区。
* 每次需求变动结束，都要考虑是否需要添加或修改测试用例。需要的话，请添加或修改测试用例。
* 每次更新PRD.md文件时，都要更新版本号，并在代码中同步该版本号（version.ts）。
* 合并新的需求时，注意不要在PRD.md中删除原有内容（除非有明确指令说原有需求被删除），但可以做补充和修正。

## 编辑文件注意事项
* 编辑代码时，注意不要在行头遗留“+”，“-”等版本控制系统的标记。以及diff带来的文件冲突标记（如<<<<<<<, =======, >>>>>>>等）。如果有，请删除这些标记。
* 每次代码编辑结束，主动进行编译，尽早发现其中的编译/语法错误等。使用npm run build命令进行编译。如获取输出失败，第二次尝试可以将输出重定向到文件中（如npm run build > output.txt 2>&1），然后查看output.txt中的错误信息。
* 修改完文件，不要提交git。

## 调试要求
* 项目调试过程中，有可能为了方便调试而添加一些临时代码，在这些临时代码的前后做上明显的标记（如注释），方便后续的删除。
* 调试相关功能在正常环境下可见可调。不要使用开发环境。
* 如要在界面上打印调试信息到窗口，请在调试窗口中提供滚动条和复制按钮，方便复制调试信息。
* 调试代码必须在调试结束后立即移除，不得留存在最终代码中。
* 为了便于快速识别和清理调试代码，请按以下规范添加标记：
  - 使用 `// DEBUG_START` 和 `// DEBUG_END` 包围调试代码块
  - 或使用 `// @DEBUG @TEMP:` 注释标记单行调试代码
  - 调试标记中应包含目的说明和日期
  - 可通过 `grep -r "@DEBUG\|DEBUG_START\|DEBUG_END" .` 快速定位所有调试代码
