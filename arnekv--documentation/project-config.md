# Metadata standards

## Required fields for all content

- `title`: Max 80 characters, content-type specific patterns
- `description`: 75-300 characters, SEO and AI optimized
- `content-type`: Must match content type being written

## Allowed values

### Audience values

Valid options (select most relevant):

- `businessUser` - Uses our products for business goals
- `administrator` - Manages cloud/on-premises environments
- `dataEngineer` - Designs data solutions
- `developer` - Builds software and systems
- `aiEngineer` - Develops AI/ML solutions
- `appMaker` - Creates low-code apps
- `dataAnalyst` - Builds data models
- `dataScientist` - Data-driven solutions
- `securityEngineer` - Digital security controls
- `solutionArchitect` - Technology integration
- `internal` - Internal emnployees only
- `partner` - Partners only
- `businessDecisionMaker` - Non-technical decision makers

### Content type values

- `concept` - Information that helps understand
- `reference` - Information to look up
- `procedure` - Information that helps do tasks
- `tutorial` - Information that helps learn
- `quickstart` - Information to get going quickly
- `troubleshooting` - Information that helps fix problems

### Experience level values

- `100` - Beginner: Learn fundamentals
- `200` - Intermediate: Build in-depth skills
- `300` - Advanced: Specialized topics

### Article type values

- `article` - Standalone article (default)
- `map` - Landing page for organization

### Lifecycle values (optional)

- `investigate` - Attract phase
- `verify` - Convert phase
- `setup` - Onboard phase
- `use` - Engage phase
- `support` - Support phase
- `transform` - Transform phase
