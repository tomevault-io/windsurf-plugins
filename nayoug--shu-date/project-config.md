# Copilot Instructions for Coding Agents

## Overview
This document provides comprehensive instructions for coding agents on how to work effectively with GitHub Copilot.

## Prerequisites
1. **GitHub Account**: Ensure that you have an active GitHub account.
2. **IDE Setup**: Install Visual Studio Code or another supported IDE with the GitHub Copilot extension enabled.
3. **Subscription**: Confirm that you have an active subscription to GitHub Copilot.

## Getting Started
1. **Enable GitHub Copilot**: Open your IDE, go to the extensions market, and enable the GitHub Copilot extension. Make sure it is authorized to access your GitHub account.

2. **Creating a New File**: Use the following steps to create a new file:
   - Open the command palette.
   - Select `New File` and name it appropriately based on your project requirements.

3. **Using Copilot**: Start typing comments or function names in your code to prompt Copilot to suggest code snippets:
   - Example:
     ```javascript
     // Function to calculate factorial
     function factorial(n) {
     ```
   - As you type, Copilot will provide suggestions. Use the `Tab` key to accept a suggestion or continue typing for more options.

## Best Practices
- **Use Clear Comments**: Provide clear comments to help Copilot generate more relevant code.
- **Iterate and Refine**: Don’t hesitate to modify the suggestions provided by Copilot. Always review the generated code for correctness.
- **Stay Updated**: Regularly check for updates to the Copilot extension and your IDE for improved performance and features.

## Troubleshooting
- If you encounter issues:
   - Ensure your IDE is connected to the internet.
   - Check the extension settings for GitHub Copilot.
   - Consult the GitHub Copilot documentation for additional support.

## Conclusion
By following these instructions, coding agents can optimize their workflow with GitHub Copilot, leading to more efficient coding and better project outcomes.
