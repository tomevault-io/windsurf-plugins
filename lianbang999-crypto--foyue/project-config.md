# UI/UX 设计规则

进行任何 UI/UX 相关工作时，**必须**遵循以下流程：

## 1. 读取设计上下文

项目根目录有 `.impeccable.md`，包含：
- 目标用户、核心用例、品牌调性
- 技术约束、差异化方向

**在动手设计前，先读取此文件。**

## 2. 激活的设计技能

本项目已安装以下技能，请主动调用：

| 技能 | 用途 |
|------|------|
| **frontend-design** | 避免 AI 通用美学，创造独特、生产级界面 |
| **Impeccable** | 排版、配色、动效、交互、响应式、UX 文案的详细准则 |
| **ui-ux-pro-max** | 风格库、配色、字体、图表、技术栈参考 |

## 3. 设计前必做

- 确认 `.impeccable.md` 中的 Design Context
- 选定明确的美学方向（如：禅意极简、古籍质感、温和现代等）
- 避免：Inter/Roboto、紫色渐变、卡片套卡片、灰色文字叠彩色背景

## 4. 设计后自检

- 通过「AI Slop Test」：如果别人一看就说「这是 AI 做的」，说明设计不够独特
- 检查 Impeccable 的 DON'T 清单，确保没有踩坑

## 5. 可用的 Impeccable 命令

需要时可使用：`/audit`（质量检查）、`/critique`（UX 评审）、`/polish`（最终润色）、`/colorize`、`/typeset`、`/arrange` 等。
