# Project: Enterprise Application Modernization

## Technology Stack
- Backend: Python 3.11+, Node.js 20+ with TypeScript
- Frontend: React 18+ with TypeScript
- API: REST (Azure APIM), GraphQL
- Database: SQLMI, MongoDB, DuckDB for analytics
- Cloud: Azure (AKS, APIM, ADX)
- Testing: Jest, Pytest, Postman, Cucumber
- CI/CD: GitHub Actions, Azure DevOps

## Code Standards
- Use TypeScript strict mode
- Follow ESLint and Prettier configurations
- Write unit tests for all business logic (>80% coverage)
- Use async/await for asynchronous operations
- Implement error handling and logging
- Follow SOLID principles

## Git Workflow
- Branch naming: `feature/JIRA-XXX-description`
- Commit message format: `JIRA-XXX: Description`
- Always create PR with proper description
- Link JIRA ticket in PR description

## Security
- Never commit secrets or API keys
- Use environment variables for configuration
- Validate all user inputs
- Implement proper authentication/authorization

## Documentation
- Add JSDoc/docstrings for all public functions
- Update README.md for significant changes
- Document API endpoints in OpenAPI format
