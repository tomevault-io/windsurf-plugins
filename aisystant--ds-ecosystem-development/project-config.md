## 4. Правила работы

### 4.1. Ops-First Rule

**При изменениях в репозитории ВСЕГДА загружай документы из 0.OPS/ ПЕРВЫМИ.**

Priority Queue:

2. `02-document-families.md` — семейства F1-F9
3. `07-naming.md` — правила именования
4. `05-glossary.md` — терминология

### 4.2. Правило нумерации X1/X2/X3

```
X1 = Надсистема (физическая вложенность)
X2 = SoI (система интереса)
X3 = Система создания
```

**НЕЛЬЗЯ** менять нумерацию в зависимости от контекста.

### 4.3. Система ≠ Эпистема

**Ядро** — только для систем (физических сущностей).

**ЗАПРЕЩЕНО:** Ядра по эпистемам (`E.Экономика/`, `G.Продвижение/`).

### 4.4. Иерархия источников знаний (Fallback Chain)

Этот репозиторий — **Downstream**. Источники знаний по приоритету:

```
1. Pack'и (source-of-truth)
   └── PACK-personal, PACK-ecosystem, PACK-digital-platform
2. SPF (форма и процесс)
   └── ~/IWE/SPF/
3. FPF (первые принципы) — только если нет в SPF
```

**Правило:** Если чего-то нет на текущем уровне — смотри выше.

**Когда читать FPF напрямую:**

- Для понимания базовых различений (A.7, A.1.1)

**Документация интеграции S2R↔FPF:** `0.OPS/0.4.FPF-Integration/`

### 4.5. Governance + Staging for Pack

> Этот репозиторий содержит governance-контент И операционную работу (семинары, предложения, черновики), которая на Close мигрирует в Pack через KE.

**Три вида контента:**

| Вид | Примеры | Жизненный цикл |
|-----|---------|----------------|
| **Governance** (постоянный) | Реестры, планы, ADR, оргструктура | Живёт здесь |
| **Операционный** (временный) | Семинары, предложения, inbox | Создаётся → работа → KE on Close → архив |
| **Pack-контент** (транзитный) | Знания домена, методы, архитектуры | Формализуется → мигрирует в Pack |

**При создании/изменении документа спроси:**
1. Governance (привязано к команде/срокам)? → Создай здесь
2. Операционная работа (семинар, предложение)? → Создай в inbox/OPS → на Close извлеки знание в Pack
3. Уже формализованное знание домена? → Сразу в Pack, здесь только ссылка

**Самоочистка:** KE on Close = механизм самоочистки. Операционный контент не накапливается — знание уходит в Pack, артефакт в архив.
