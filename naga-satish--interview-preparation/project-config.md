## Why (Purpose)

This repository helps prepare for technical interviews by providing comprehensive, interview-focused answers to common questions across the modern data and infrastructure stack. Answers are designed to be:
- **Concise**: 50-200 words (interview-appropriate length)
- **Practical**: Focus on real-world usage and best practices
- **Structured**: Use bullets, tables, code blocks for clarity
- **Current**: Reference latest features and official documentation

## How (Working on This Repository)

### Answer Format Standard

Each file follows this structure:
- Table of Contents with numbered sections
- Questions numbered within sections: `### 1. Question text?`
- **One blank line** after each question
- Answer starts with `**Answer:**` prefix
- **No indentation** (standard markdown)
- Reference: [Apache-Airflow.md:27-36](Apache-Airflow.md#L27-L36)

**Example:**
```markdown
### 1. Question text here?

**Answer:** Clear opening statement explaining the core concept.

- Key point 1
- Key point 2
- Key point 3

**Example:** [if it clarifies the concept]
```

### Answer Guidelines

**Format:**
- Start with `**Answer:**` prefix
- Target 50-200 words (brief but complete)
- Use bullet points, tables, or code blocks
- Add sub-sections for complex topics: **Key Properties:**, **Use Cases:**, **Trade-offs:**
- Use backticks for code elements: `dag_id`, `SELECT`, `kubectl`

**Content:**
- Directly answer the question
- Include practical examples or use cases
- Explain technical terms clearly
- Focus on interview-relevant information
- Verify against official documentation

### Quality Checklist

Before submitting answers:
- ✓ Directly answers the question
- ✓ 50-200 words (concise but complete)
- ✓ Includes practical examples
- ✓ Technical terms clearly explained
- ✓ Structured for easy scanning
