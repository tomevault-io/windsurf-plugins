# Research Paper / Garden Post Review Checklist

**Use this before publishing any research-style content.**

## Goal

Catch weaknesses in:
- argument clarity
- factual accuracy
- source quality
- citation hygiene
- epistemic honesty
- narrative structure
- prose quality
- publication risk

This rule aligns with the [blogging guidelines](.cursor/rules/blogging.mdc) (especially citation rules) and the site's emphasis on rigorous, honest technical writing.

## Review Stages

### 1. Thesis Pass
- What is the core question/claim?
- Is the main answer stated early, in plain English?
- Does the title match the actual claim?
- Is the scope narrow enough to defend convincingly?

### 2. Claim Inventory Pass
List every nontrivial claim and classify:
- Fact
- Interpretation
- Speculation
- Rhetorical framing

**Checks**:
- No mixing fact and speculation in the same sentence.
- No claims stronger than the evidence.
- Every throwaway factual claim has a citation.

### 3. Source Audit Pass
For each important factual claim:
- What is the *primary* source?
- Did I read it directly (not via another blog)?
- Is it the best available source?
- Is it stable/archived?
- Do I have an archive link?

**Red flags**:
- Citation laundering (blog → blog).
- Wikipedia as sole authority on contested claims.
- Using a source for more than it says.
- Relying on memory.

### 4. Evidence Chain Pass
- Can a skeptical reader reconstruct the reasoning?
- Is every inferential step visible?
- Have I quoted/summarized the decisive evidence?
- Have I addressed the strongest alternative readings?

For synthesis across sources, make the synthesis explicit.

### 5. Uncertainty Pass
Clearly separate:
- Well-established
- Plausible
- Inferred
- Unknown

**Checks**:
- Label uncertainty honestly.
- Never imply certainty where you have only plausibility.
- Consider moving weak-but-interesting material to an appendix.

### 6. Citation Hygiene Pass (Blog-Specific)
Follow the exact rules from `blogging.mdc`:
- Prominent first introduction (anchor link) when a source is a *subject of discussion*.
- Footnotes for later supporting claims.
- Use markdown reference links to DRY when a source has both intro and footnote.
- Exact citation target must match the claim.
- No ornamental citations.

**Checks**:
- Anchor links and footnotes serve distinct jobs.
- No overloaded citation paragraphs (prose should carry the logic).
- No uncited claims riding on a nearby mentioned source.

### 7. Structure Pass
- Does the intro answer the motivating question quickly?
- Does every section advance the argument?
- Are digressions contained or cut?
- Does the conclusion return cleanly to the original question?

**Checks**:
- Could two adjacent sections be merged?
- Is any section present only because "the research was interesting"?

### 8. Counterargument / Hostile Reader Pass
Try to break the piece. Ask:
- What would the smartest critic dispute first?
- Which claim is hardest to defend?
- Where am I over-reading sparse evidence?
- What competing explanation did I ignore?
- What would a domain expert call sloppy?

**Fix the strongest objection before publishing.**

### 9. Prose Pass
- Precise rather than inflated?
- Confident tone without pretending to certainty?
- Concrete terms, exact nouns/dates/names/mechanisms?
- No filler, no throat-clearing, no "scholarly cosplay"?

Cut anything that merely *sounds* rigorous.

### 10. Publication Hygiene Pass
- Consistent names, dates, titles.
- All links work (including archives).
- Quotes are exact.
- Formatting clean.
- Sensitive/local identifiers redacted.
- Section headings earn their keep.

## Scoring Rubric (1–5 per axis)

- Argument clarity
- Source quality
- Claim support
- Epistemic honesty
- Citation hygiene
- Structure
- Readability
- Reproducibility
- Original value
- Risk

**Hard gate**: Do not publish if `source quality`, `claim support`, or `epistemic honesty` < 4.

## Practical Workflow

1. Read once for argument only.
2. Build a claim ledger.
3. Audit every important claim against primary sources.
4. Rewrite overstated or weakly-supported passages.
5. Clean up citations per blogging rules.
6. Run the hostile-reader pass.
7. Publish only if it still reads smoothly.

## Fast Pre-Publish Questions (when short on time)

1. Core answer in one sentence?
2. What are the three most important claims and their best sources?
3. Where am I least certain?
4. What would the strongest critic say?
5. Have I made any claim more confident than my evidence supports?
6. If a reader checks my sources, will they feel misled?
