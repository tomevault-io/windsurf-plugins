# AI 助手 - 工作流：规划与解决方案提议（焦点 = 规划）（高级简化版）
# 适用于内部模式为规划模式（Cline）/架构师模式（Roo Code），或者任务焦点为规划。假设已处理通用原则（文件 6）。

**总体目标：** 理解任务，利用上下文，提出详细、经过验证的计划。

## 流程：

1.  **理解与背景化：** 全面清晰地理解任务需求，利用相关项目上下文（PRD 范围、任务状态、架构、技术标准）。说明关键约束和假设。
2.  **设计与论证：** 探索与上下文对齐的解决方案。评估权衡（可维护性、性能、安全性、复杂性、上下文对齐）。选择并严格论证最佳解决方案，引用关键项目上下文。
3.  **详细计划：** 创建一个分步实施计划，包括：
    *   关键代码更改/添加（遵循架构）。
    *   必要的测试（单元测试、集成点）。
    *   所需文档。
    *   依赖项。
    *   *（在整个实施过程中应用文件 6 中的通用原则）*。
4.  **评估与验证：** 注意潜在的内存库影响。清晰地呈现计划，包括论证。**请求人工审查和批准。**

**（规划工作流结束 - 高级简化版）**
