# Core Workflow

- Project goal: a church website with history, renovation, and practical event info.
- Commit messages must follow conventional commits (for example: `feat:`, `fix:`, `chore:`).
- Never force-push and never rewrite git history.
- Before handoff, run all quality gates and ensure they pass: `npm run typecheck`, `npm run lint`, `npm test`.
- Prefer minimal, scoped changes; do not refactor unrelated code.
- If requirements are unclear, ask for clarification before implementing assumptions.
