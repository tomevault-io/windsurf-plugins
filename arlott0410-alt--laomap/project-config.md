# GitHub push after code work

When you finish implementing or fixing code in this repository:

1. Run `git status` and ensure intended files are staged.
2. Commit with a clear message describing what changed.
3. Push to `origin` on branch `main`: `git push origin main` (or `git push` if upstream is set).

Do this as part of the same session as the code changes unless the user explicitly asks to skip push or there is no remote access.
