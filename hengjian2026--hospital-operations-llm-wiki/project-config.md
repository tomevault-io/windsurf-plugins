# CLAUDE.md — 医院运营知识库

## 概述

这是一个关于**医院运营管理数据分析**的个人知识库，基于 LLM Wiki 模式构建。知识库汇集了医院各科室的 DRG/DIP 病种分析、绩效考核、财务分析等报告，通过 LLM 增量维护跨文档的关联和洞察。

## 目录结构

```
older/
├── CLAUDE.md           ← 本文件：schema 定义
├── index.md            ← 知识库总索引
├── log.md              ← 操作日志（按时间记录）
├── raw/                ← 原始文档（只读）
│   ├── doc/            ← .doc 文件

│   ├── 实体/           ← 科室、疾病、指标等实体
│   ├── 方法论/         ← 分析框架（DRG分析、DIP分析、绩效评估等）
│   ├── 来源/           ← 原始文档摘要页

└── .obsidian/          ← Obsidian 配置
```

## 核心概念

### 分析方法论
- **描述性统计**：均值、中位数、极值、分布、异常值识别
- **相关性分析**：相关性检验、偏相关、散点图拟合
- **回归分析**：线性回归、逻辑回归，定量影响因素
- **机器学习**：决策树、深度学习，用于预测和分类
- **控制图**：数据驱动的质量指标监控
- **波士顿矩阵**：科室/病种竞争力分析
- **帕累托图**：主要因素分析（二八法则）
- **森林图**：多维度比较分析

### 核心指标体系
- **门诊指标**：门诊人次、挂号数、门诊收入
- **住院指标**：出院人次、住院天数、床位使用率
- **手术指标**：手术量、日间手术率、微创手术率、四级手术率
- **DRG/DIP 指标**：CMI、盈亏金额、病种倍率、组内变异系数
- **绩效指标**：目标完成率、质量安全指标、费用控制
- **财务指标**：医疗收入结构、成本构成、人力成本

### 主要分析维度
- **科室维度**：各临床科室运营分析（呼吸科、胸外科、儿科等）
- **病种维度**：按 DRG/DIP 分组的病种盈亏分析
- **时间维度**：年度趋势、季度跟踪、月度监控
- **费用维度**：药占比、耗占比、住院费用结构

## Wiki 维护流程

### Ingest（摄入新文档）

当用户将新的 docx 放入 `raw/docx/` 并要求摄入时，执行以下步骤：

1. **读取原始文件** — 用 Python 提取 docx 中的纯文本内容
2. **分析内容** — 识别：主题类型、核心发现、关键数据、涉及的科室/病种/指标
3. **创建来源页** — 在 `wiki/来源/` 下创建 `YYYYMMDD-文档标题.md`，包含：
   - 来源信息（原始文件名、日期、主题）
   - 核心内容摘要（分要点）
   - 关键数据表格（如有）
   - 可视化图表列表
   - 相关 [[概念]]、[[实体]]、[[方法论]] 的内部链接
4. **更新相关实体/概念页** — 如果来源引入了新科室、新指标、新方法，在对应页面补充内容并添加返回来源的链接
5. **更新 index.md** — 在对应分类下添加一行（来源名 + 1行简介）
6. **记录到 log.md** — 格式：`## [日期] ingest | 文档标题`，附简短说明

### Query（查询）

当用户提出问题时，执行以下步骤：

1. **搜索相关页面** — 先读 `index.md` 的分类索引，或直接用关键词搜索 wiki 页面
2. **综合信息** — 读取 3-5 个最相关的 wiki 页面，综合回答
3. **生成答案** — 引用相关页面的 `[[链接]]`，提供结构化回答
4. **归档价值结论** — 如果用户的问题产生了新的洞察或综合结论，主动创建 `wiki/报告/新页面.md` 归档

### Lint（健康检查）

每摄入 5 个新来源，或用户要求检查时，执行：

1. **孤立页面检查** — 搜索没有被任何页面引用的 wiki 页面（无入链）
2. **矛盾检查** — 同一指标在不同来源页中的数据是否一致
3. **过时检查** — 新摄入的来源是否覆盖了旧页面的结论（如有矛盾，在旧页面注明"被 [[来源/新文档]] 更新"）
4. **缺失链接检查** — 来源页是否链接了相关概念/实体/方法论
5. **orphan 来源页** — 是否存在没有对应原始文件的来源页

报告给用户，让用户决定是否修复。

## 命名规范

- 页面标题：使用中文，简洁明了
- 文件名：`kebab-case` 小写连字符
- 内部链接：`[[页面标题]]`
- 标签：`#标签名` 放在 YAML frontmatter

## Metadata (元数据)

所有 Wiki 页面（尤其是 `来源/`、`实体/`、`概念/`、`方法论/` 等）必须在文件顶部包含标准的 YAML frontmatter 块，以便 Dataview 插件能生成动态表格和列表。格式如下：

```yaml
---
title: 页面标题
type: 页面类型 (如: 来源, 实体, 概念, 方法论)
tags: [标签1, 标签2]
date: YYYY-MM-DD (仅来源页或报告页需要)
---
```

## 状态追踪

- `index.md` 中的所有页面需有 1 行简介
- `log.md` 记录每次摄入和重大更新
- 定期运行 Lint 确保 wiki 健康度
