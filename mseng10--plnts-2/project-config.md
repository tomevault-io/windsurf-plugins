## Project Overview

This is a home system management application (possibly for plant care) with both frontend and backend components.

## Architecture

### Backend (Python/FastAPI)
- Built with FastAPI and MongoDB for data persistence
- Uses FastMCP for MCP (Model Control Protocol) integration
- Has a comprehensive set of routes for various entities (plants, systems, todos, mixes, stats, alerts, etc.)
- Uses a custom entrypoint script that waits for MongoDB to be ready
- Has a scheduler for background tasks

### Frontend (React)
- Built with React 18, Tailwind CSS (dark mode), Framer Motion, Lucide React, Day.js
- Uses Create React App
- Has a comprehensive API layer defined in `client/src/api.js`
- Uses custom hooks for state management (no Redux or Context)
- Has a specific structure with pages, modals, and routing

### Docker Setup
- Multi-service docker-compose with app, mongo, client, and nginx
- Dev mode: The app service runs on port 8002, client on 8080 (no TLS)
- Prod mode: App runs on SSL/uvicorn, nginx listens on 8443 for HTTPS (with HTTP redirect), MongoDB uses TLS
- Production overlay (`docker-compose.prod.yml`) wires TLS into all services
- A `tls-init` init container generates self-signed CA and per-service certificates, stored in `tls_certs` volume
- Services wait for `tls-init` via `service_completed_successfully` before starting

## Key Components

### Backend Services
- `server/app/app.py`: Main FastAPI application with routes and database initialization
- `server/mcp_server/server.py`: MCP server setup
- `server/app/entrypoint.sh`: Entry point script that waits for MongoDB to be ready
- `server/requirements.txt`: Python dependencies including FastAPI, MongoDB driver, FastMCP, etc.

### Frontend Services
- `client/src/api.js`: API layer with endpoint definitions and HTTP helpers
- `client/Dockerfile`: Frontend Dockerfile for building and running the React app

## API Endpoints

All backend endpoints are accessible through the FastAPI server running on port 8002. The frontend communicates with the backend using the API endpoints defined in `client/src/api.js`.

## Database

- MongoDB 7.0 is used for data persistence
- Two databases are configured: `plnts` and `plnts_hist`
- MongoDB is initialized with an admin user

## TLS & Security

### Production Deployment (TLS Enabled)
- All services communicate via TLS in production mode
- A self-signed CA and per-service certificates are generated by the `tls-init` init container
- Certificates are stored in a Docker volume (`tls_certs`) and persist across container restarts
- Rolling upgrades (e.g., `make restart-app`) preserve existing certificates

### Certificate Generation & Management
- `tls/generate-certs.sh`: Generates a self-signed CA and certificates for app, nginx, and mongo
- Skips generation if certs already exist (supports rolling upgrades)
- Creates Subject Alternative Names (SANs) matching Docker DNS names for each service
- Key files are stored with restrictive permissions (mode 600)

### Service Configuration
- **App (uvicorn)**: `server/app/entrypoint.sh` passes `--ssl-keyfile` and `--ssl-certfile` when `TLS_ENABLED=true`
- **Nginx**: Uses `nginx/nginx.prod.conf.template` which configures HTTPS on port 8443, applies TLS hardening (HSTS), and verifies upstream SSL to app
- **MongoDB**: Connection string includes TLS parameters when in production mode

## Development Commands

### Dev Mode (no TLS)
```bash
# Start the application
make dev

# Build and start
make dev-build

# View logs
docker-compose logs -f

# Stop the application
docker-compose down
```

### Prod Mode (with TLS)
```bash
# Start with TLS between all services
make prod

# Build and start with TLS
make prod-build

# View logs (works with both dev and prod)
docker-compose -f docker-compose.yml -f docker-compose.prod.yml logs -f

# Rolling upgrades (preserve TLS certs)
make restart-app   # Redeploy app only
make restart-nginx # Redeploy nginx only

# Force certificate regeneration
make regen-certs

# Stop all services (works with both modes)
make down
```

## Environment Variables

### General
- `MONGODB_URL`: MongoDB connection string for main database
- `MONGODB_URL_HIST`: MongoDB connection string for history database
- `REACT_APP_API_URL`: API base URL for frontend (dev: `http://localhost:8002`, prod: `https://localhost:8443`)

### TLS (Production Only)
- `TLS_ENABLED`: Set to `true` in production mode to enable SSL/TLS for uvicorn
- Certificates are auto-generated in `/certs` volume via `tls-init` init container and reused across restarts for rolling upgrades
