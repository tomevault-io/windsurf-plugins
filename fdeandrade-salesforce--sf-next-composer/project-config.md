# UI and Styling Standards

## Component Library

- **Radix UI**: Use Radix UI primitives for accessible components
- **Shadcn Components**: Our presentational components use Shadcn as a styling layer on top of Radix Primitives and are located in `src/components/ui/`
- **Icons**: Use Lucide React and React Simple Icons

## Styling Approach

- **Tailwind CSS v4**: Use utility-first CSS framework
- **CSS Classes**: Use the `cn()` utility for conditional classes
- **Responsive Design**: Implement mobile-first responsive design
- **Accessibility**: Ensure proper ARIA labels and keyboard navigation

## Component Patterns

- **Composition**: Use compound component patterns where appropriate
- **Props Interface**: Define clear prop interfaces with TypeScript
- **Default Values**: Provide sensible defaults for optional props
- **Error States**: Implement proper error and loading states

## Design System

- Follow consistent spacing, typography, and color schemes
- Use semantic HTML elements
- Implement proper focus states and hover effects
- Ensure WCAG compliance for accessibility
