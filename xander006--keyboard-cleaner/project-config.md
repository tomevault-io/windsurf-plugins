## Working Principles

**Plan before coding.** Outline the approach, architecture, and key steps before writing any code. Wait for confirmation if the scope is unclear.

**Preserve working state.** Create a backup before structural or risky changes. Never refactor, migrate, or modify critical logic without preserving the current working state first.

**Done means tested.** Code must compile, run, and pass relevant tests before being considered complete.

**Start minimal.** Implement the smallest viable solution first, then iterate only if needed. Solve the current problem, not hypothetical future ones.

**Make assumptions explicit.** State assumptions clearly before proceeding when something is unclear.

**Prefer readability over abstraction.** Use straightforward logic. Only abstract when repetition or complexity clearly justifies it.

**Fail fast and visibly.** Avoid silent failures. Errors should be explicit and easy to diagnose.

**Keep dependencies minimal.** Don't introduce new libraries unless strictly necessary.

**Comment intent, not the obvious.** Explain why something exists, not what the code literally does.
