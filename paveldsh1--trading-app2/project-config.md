# Trading App - Правила проекта

## Технические предпочтения

### Именование
- Компоненты React: PascalCase (AuthForm, ChartComponent)
- Файлы компонентов: kebab-case (auth-form.tsx, chart-component.tsx)
- Файлы страниц: kebab-case (page.tsx, layout.tsx)
- Функции: camelCase (handleAuth, fetchHistoricalData)
- Переменные: camelCase (isLoading, selectedSymbol)
- CSS классы: kebab-case с использованием TailwindCSS классов

### Структура файлов
- Компоненты размещаются в src/components/
- Страницы в src/app/
- Backend API в src/api/
- Стили в src/app/global.css и через TailwindCSS

### Работа с API
- Использовать fetch для HTTP запросов
- WebSocket для получения данных в реальном времени
- Обрабатывать ошибки через try-catch и отображать понятные пользователю сообщения
- Сохранять SSID в localStorage для персистентности сессии

### UI/UX
- Темная цветовая схема (темно-синий фон, белый текст)
- Основной цвет: зеленый (#16a34a)
- Вторичный цвет: розовый (#db2777)
- Адаптивный дизайн с использованием TailwindCSS grid и flex
- Индикаторы загрузки при асинхронных операциях
- Четкое отображение статуса соединения

## Паттерны реализации

### Авторизация
- Сохранение SSID в localStorage
- Проверка валидности SSID через запрос к API
- Автоматическое восстановление сессии при перезагрузке страницы

### Работа с графиками
- Использование Lightweight Charts для отображения графиков
- Загрузка исторических данных при инициализации
- Подключение к WebSocket для обновлений в реальном времени
- Обработка различных форматов данных (поддержка time как string и number)
- Отображение статуса соединения в углу графика

### Обработка ошибок
- Try-catch блоки для всех асинхронных операций
- Отображение ошибок в UI с понятными сообщениями
- Логирование ошибок в консоль с подробной информацией
- Автоматические попытки переподключения при потере соединения

## Требования к коду

### TypeScript
- Строгая типизация для всех компонентов и функций
- Интерфейсы для определения структуры данных
- Использование необязательных параметров с дефолтными значениями

### React
- Функциональные компоненты с хуками
- Разделение компонентов по принципу единой ответственности
- Выделение повторяющейся логики в отдельные хуки
- Избегание лишних ререндеров

### Производительность
- Ограничение количества данных на графике
- Оптимизация обработки данных WebSocket
- Мемоизация тяжелых вычислений
- Оптимальное использование React.memo() для предотвращения лишних ререндеров
