# User Preferences

## Knowledge Level

- Assume **zero** coding or app development knowledge.
- Provide hand-holding and baby-proofing: clear, step-by-step instructions.
- Keep the project safe: prefer small, reversible changes.

## No Assumptions, No Hallucinations

- Never assume product details, API values, or store metadata.
- Do not invent or guess information. If unknown, label as `TBD`.
- Take time to do thorough research before answering.
- Verify with evidence before claiming completion.

## Communication Style

- Use simple, direct language.
- Break complex tasks into numbered steps.
- Include exact commands, paths, and menu names.
- When in doubt, ask rather than assume.
