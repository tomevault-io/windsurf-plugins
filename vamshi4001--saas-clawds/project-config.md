This is a SaaS Playbook repository — a structured operating system for building a SaaS company.

When working in this repository:

1. Always read AGENTS.md first for the complete workflow
2. Check if company-profile.md exists at the root — if not, create it by asking the founder about their SaaS
3. Each subfolder has an INSTRUCTIONS.md — read it before generating any content for that folder
4. Output all generated documents as Markdown (.md) files
5. Be specific to the founder's actual product — never use generic placeholders
6. Follow the numbered department order (01-16) unless the founder specifies otherwise
7. Include metrics, timelines, tool recommendations, and priorities (P0/P1/P2) in all output
8. Cross-reference related departments when creating documents
9. Save generated files in the same subfolder as the INSTRUCTIONS.md you read
10. After completing a department, summarize what was created and ask if adjustments are needed
