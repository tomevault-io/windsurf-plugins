# GitHub Copilot 仓库级指令

本仓库提供一套面向多 AI 编码助手的可维护配置体系。你在本仓库内工作时，必须优先遵守以下约定：

## 工作方式

- 先阅读相关代码、配置和脚本，再下结论。
- 优先做能解决问题的最小改动。
- 优先复用现有模式，再考虑新增抽象。
- 不主动扩需求；只有在安全完成当前任务必须时才扩大改动范围。

## 质量要求

- 将错误行为、类型问题、缺失校验、缺失错误处理、明显回归视为真实问题。
- 不通过修改测试来掩盖错误实现。
- 改动后做受影响范围内最小但有意义的验证。

## 协作与输出

- 默认使用简体中文回复；代码、命令、日志、报错和协议字段保持原文。
- 默认保持专业、直接、低噪音表达。
- 文档和代码中的作者署名统一使用 `wwj`；如果需要补充作者信息，不使用其他署名。
- Java 代码文件在新增或补齐文件头注释时，头部作者注释统一使用 `@author wwj`。

## 分层

- 语言和框架规范优先参考项目内现有配置目录，例如 `.codex/skills/`、`.claude/skills/`、`.cursor/skills/` 中对应语言规范。
- 对于修复、需求、设计、提交流程等任务，优先复用本仓库既有 workflow，而不是临时发明新的流程。
