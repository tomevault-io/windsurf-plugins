# GEMINI.md: Building Embedded Firmware for ESP32 with ESP-IDF

This document serves as a comprehensive guide for setting up, building, and deploying embedded firmware for the ESP32 microcontroller using the official **Espressif IoT Development Framework (ESP-IDF)**.

---

## 1. Prerequisites & Required Tools

Before starting, ensure you have the following hardware and software installed.

### Hardware
* **ESP32 Development Board** (e.g., ESP32-DevKitC, ESP-WROOM-32).
* **USB Cable** (Micro-USB or USB-C, depending on the board) capable of data transfer (not just charging).



### Software Dependencies
* **Git:** Required for cloning the ESP-IDF repository.
* **Python (3.8+):** Required for the build tools (`idf.py`).
* **CMake:** The build system used by ESP-IDF.
* **Ninja:** A small build system with a focus on speed.
* **Drivers:** CP210x or CH340 USB-to-UART drivers (depending on your specific board's USB-to-Serial chip).

---

## 2. Installation & Setup (Recommended Method)

The most efficient way to develop is using **Visual Studio Code (VS Code)** with the official extension.

### Step 1: Install VS Code
Download and install from [code.visualstudio.com](https://code.visualstudio.com/).

### Step 2: Install the Espressif IDF Extension
1.  Open VS Code.
2.  Go to the **Extensions** view (Ctrl+Shift+X).
3.  Search for **"Espressif IDF"**.
4.  Click **Install**.

### Step 3: Configure the Extension
1.  Press `F1` or `Ctrl+Shift+P` to open the Command Palette.
2.  Type and select **ESP-IDF: Configure ESP-IDF extension**.
3.  Select **Express** installation.
4.  Follow the wizard to download the ESP-IDF framework, tools, and Python virtual environment.

> **Note:** This process creates a dedicated `.espressif` folder (usually in your home directory) containing the toolchain and the specific version of ESP-IDF.

## 3. Project Structure

An ESP-IDF project differs from Arduino. It uses CMake. A standard project looks like this:

```text
my_esp32_project/
├── CMakeLists.txt          # Top-level CMake file for the project
├── sdkconfig               # Project configuration (generated by menuconfig)
├── main/                   # Source code folder
│   ├── CMakeLists.txt      # Component CMake file
│   └── main.c              # Entry point of your application
└── components/             # (Optional) Custom libraries/components
