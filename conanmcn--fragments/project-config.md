# Landmines

- Repo folder named `segments/` but branding is "Fragments" — Historical name. All npm packages, docs, CLI use "fragments". Renaming would break paths everywhere
- 8 `eslint-disable react-hooks/exhaustive-deps` (`libs/ui/src/components/NavigationMenu/index.tsx`) — Complex keyboard navigation requires manual dep control in effects. Not a bug
- `@ts-expect-error` on virtual module imports (`packages/cli/src/viewer/entry.tsx`) — Vite virtual modules have no TS declarations. Runtime-resolved by Vite plugin
- `@ts-expect-error` on Vite HMR event types (5 instances) (`packages/cli/src/viewer/hooks/useHmrStatus.ts`, `preview-frame-entry.tsx`) — Vite internal HMR events are not typed in public API. Used for dev UX
- Lazy `require()` with `eslint-disable` for optional peer deps (8 components) (`libs/ui/src/components/CodeBlock/index.tsx` + 7 others) — Components check availability at runtime and fail gracefully. Also in DatePicker, Chart, Markdown, ColorPicker, DataTable, Editor
- Constants duplicated between CLI and MCP packages (`packages/mcp/src/constants.ts`) — Intentionally inlined to avoid MCP pulling in full CLI dependency. Comment in file explains this
- Legacy file extensions still supported (`.segment.tsx`, `segments.config.ts`, `.recipe.ts`) (`packages/cli/src/core/constants.ts`) — Migration path for early adopters. CLI file discovery checks both old and new extensions
- 3 components with upstream Base UI waivers (Menu, Select, Combobox) (`e2e/auditor/waivers.ts`) — Tracked in COMPONENT_WAIVERS registry. Upstream Base UI focus guard patterns, expires 2026-06-30
- jsdom canvas errors in test output (`libs/ui/src/components/` stderr) — axe-core tries to measure canvas elements in jsdom. Harmless — tests still pass correctly
- Convex generated files with `eslint-disable` (`packages/mcp/convex/_generated/`) — Auto-generated by Convex. Never edit manually
- `as const satisfies` makes optional fields non-existent — helper functions use type assertions to access meta/shift/alt (`libs/ui/src/utils/keyboard-shortcuts.ts`)

# Decisions

- Base UI headless primitives for component foundation (over: Radix UI, Headless UI, Ariakit) — Unstyled primitives with full ARIA, custom styling via SCSS modules + tokens
- SCSS Modules + CSS custom properties (over: styled-components, Emotion, Tailwind, vanilla-extract) — Dual fallback `var(--fui-token, $fui-token)` for compile-time and runtime theming. No JS runtime cost. SSR-safe
- 5-seed theming system deriving 120+ tokens (over: Per-token configuration, theme objects) — 5 values (`$fui-brand`, `$fui-neutral`, `$fui-density`, `$fui-radius-style`, `$fui-danger`), system derives everything
- Fragment files as machine-readable component metadata (over: Storybook MDX, JSDoc) — Colocated with code, JSON-serializable for AI, runtime-validated, compiled to fragments.json
- Monorepo with Turborepo + pnpm workspaces (over: Separate repos, Nx, Lerna) — Shared types, atomic commits, single CI with caching
- Hybrid search with Reciprocal Rank Fusion (k=60) (over: Pure keyword, pure vector) — RRF merges keyword + vector ranked lists robustly across different score ranges
- Two-tier CLI exports (cli vs cli/core) (over: Single package, separate packages) — cli/core is browser-safe. Full cli needs Node.js. Docs app imports core without Node deps
- Object.assign compound component pattern (over: React Context per sub-component, render props) — Zero-overhead composition. Card.Header discoverable via dot notation. TypeScript infers sub-component props
- MCP tool factory pattern (buildMcpTools) (over: Duplicate tool definitions per server) — Single source of truth for 12 MCP tools. Both CLI and standalone MCP use same factory
- Optional peer deps for heavy libraries (over: Bundle everything, separate packages) — DatePicker, Chart, CodeBlock etc use lazy require(). Core bundle stays small
- Dual license: UI is MIT, tooling is FSL-1.1-MIT (over: All MIT, all proprietary) — UI MIT for adoption. Tooling FSL — free for non-competing use, converts to MIT after 2 years

# Conventions

## Patterns
- **Component structure**: Each component is a folder with `index.tsx` + `.module.scss` + `.test.tsx` + `.fragment.tsx`
- **Compound components**: `Object.assign(Root, { Sub: ... })` pattern for `Card.Header`, `Dialog.Body`, etc. Used by 63/67 components
- **Barrel exports**: `libs/ui/src/index.ts` re-exports main + Root + all sub-components + all prop types per component
- **SCSS dual fallback**: `var(--fui-token, $fui-token)` mandatory in all component styles — CSS vars for runtime theming, SCSS vars for build-time
- **Token prefix**: All tokens use `fui-` prefix (e.g., `--fui-color-accent`, `$fui-space-2`)
- **Seed-based theming**: 5 seeds derive 120+ CSS custom properties
- **Fragment files**: `defineFragment({ meta, usage, props, relations, contract, variants })` from `@fragments-sdk/cli/core`
- **Block files**: `defineBlock({ name, description, category, components, tags, code })` from `@fragments-sdk/cli/core`
- **Test pattern**: `vitest` + custom `render`/`screen`/`userEvent`/`expectNoA11yViolations` from `../../test/utils`
