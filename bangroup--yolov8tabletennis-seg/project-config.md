# Контекстные капсулы (RLM)

## Иерархия источников (строгая)
```
Данные/Конфиги (SSoT) → Капсулы (контекст для агентов)
```
Капсулы — **read-only проекции**, не SSoT. При расхождении — верить SSoT.

## Манифест
- `config/capsules_manifest.json` — индекс всех капсул
- Каждая запись: `id`, `path`, `domain`, `covers`, `updated`
- **Перед началом работы** прочитай манифест для навигации по контексту

## Формат капсулы (8 обязательных секций)
1. **Scope** — что трогаем / не трогаем
2. **Invariants** (≤12) — со ссылкой на SSoT
3. **Decisions** (≤7) — ключевые архитектурные решения
4. **Impact Paths** — граф влияния
5. **Validation Proof** — как проверяется
6. **Risks** (≤7) + Mitigations
7. **Open Questions** (≤7)
8. **Pointers** (≤15) — ссылки на файлы

## Правила для агентов
- **Все агенты**: читать манифест при старте задачи → выбирать 1-2 капсулы → работать в фокусе
- **capsule-builder**: создаёт/обновляет капсулы + обновляет манифест
- **Оркестратор**: может обновлять капсулы напрямую; при делегировании — указывать какие капсулы читать

## Изменение капсул
1. Изменение фактов → правим SSoT → обновляем капсулу
2. Новая капсула → capsule-builder создаёт по шаблону → обновляет манифест
3. Governance-гейт не требуется (капсулы — производные)
